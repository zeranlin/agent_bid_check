# 政府采购招标文件合规审查报告

**项目名称**：深圳市宝安区信息中心宝安区民生诉求服务平台（二期）项目
**项目编号**：BACG2025000066
**审查人**：政府采购招标文件合规审查人
**审查日期**：2025 年 X 月 X 日

---

## 一、审查综述
经对招标文件全文进行逐项审查，发现该文件在**资格条件设置、评分标准量化、技术参数倾向性、商务条款公平性**等方面存在多处高风险及中风险问题。部分条款涉嫌以不合理的条件限制或排斥潜在供应商，部分评分标准主观性过强且缺乏量化依据，部分业绩和证书要求与项目实际需求关联度存疑，存在指向特定供应商的倾向。

---

## 二、详细风险点清单

### 1. 资格条件设置不合理（高风险）

| 项目 | 内容 |
| :--- | :--- |
| **问题标题** | 设置与项目履约能力无关的特定行业认证及企业性质限制 |
| **问题定性** | **高风险** |
| **审查类型** | 资格条件/限制竞争 |
| **原文位置** | 第一册 专用条款 第一章 招标公告 申请人的资格要求 第 11、12 条 |
| **原文摘录** | "11. 投标人须具备保安服务认证证书；<br>12. 投标人须具备全国科技型中小企业证书；” |
| **风险判断** | 1. **保安服务认证证书**：本项目为“民生诉求服务平台”软件开发及运维服务，属于信息技术服务，与“保安服务”无直接关联。强制要求具备保安服务认证证书，属于以不合理的条件限制或排斥潜在供应商，涉嫌指向特定具有保安资质的企业。<br>2. **全国科技型中小企业证书**：虽然鼓励科技创新，但将其作为**强制性资格条件**（即不具备则投标无效），而非评分项，违反了《政府采购法实施条例》第二十条关于不得以特定行政区域或特定行业的业绩、奖项作为加分条件或中标条件的精神，且该证书并非软件开发类项目的法定准入资质。 |
| **法律/政策依据** | 《中华人民共和国政府采购法实施条例》第二十条第（三）项：“不得以不合理的条件对供应商实行差别待遇或者歧视待遇……（三）采购需求中的技术、服务等要求指向特定供应商、特定产品……"；《政府采购货物和服务招标投标管理办法》（财政部令第 87 号）第十七条。 |
| **整改建议** | 1. 删除“投标人须具备保安服务认证证书”的资格要求。<br>2. 将“全国科技型中小企业证书”由资格条件调整为商务评分项（如：具备该证书得 X 分），或删除该要求。 |

### 2. 评分标准主观性过强，缺乏量化依据（高风险）

| 项目 | 内容 |
| :--- | :--- |
| **问题标题** | 技术方案评分标准中“优、良、中、差”档次划分模糊，自由裁量权过大 |
| **问题定性** | **高风险** |
| **审查类型** | 评分办法/评分因素不相关 |
| **原文位置** | 评标信息 技术部分 评分准则（实施方案、重点难点、质量保障、售后服务、智能应用、民意速办等所有方案类评分项） |
| **原文摘录** | “在此基础上，由评审委员会对具体响应内容进一步评审：<br>①评审为优（方案符合本项目实际需求，方案内容合理、条理清晰、可操作性强）的最高得 40 分;<br>②评审为良……最高得 20 分;<br>③评审为中……最高得 10 分;<br>④评审为差……得 0 分。” |
| **风险判断** | 1. **标准模糊**：“合理”、“清晰”、“可操作性强”等词汇缺乏客观量化标准，完全依赖评委主观判断，极易导致评分结果不公。<br>2. **分值跨度大**：从“良”到“优”直接跨越 20 分，缺乏中间过渡的细化标准，容易引发质疑投诉。<br>3. **违反量化原则**：综合评分法要求评审因素应当量化，此类定性描述过多的评分标准不符合《政府采购货物和服务招标投标管理办法》第五十五条关于“评审因素应当细化和量化”的要求。 |
| **法律/政策依据** | 《政府采购货物和服务招标投标管理办法》（财政部令第 87 号）第五十五条：“评审因素应当细化和量化，且与相应的商务条件和采购需求对应……不得将评审因素细化为‘优、良、中、差’等模糊档次。” |
| **整改建议** | 1. 废除“优、良、中、差”的定性评分方式。<br>2. 将评分标准细化为具体的量化指标。例如：针对“可操作性”，可规定“提供详细的项目进度甘特图得 5 分，提供关键路径分析得 5 分，提供风险应对预案得 5 分”等，确保每项得分都有明确的依据。 |

### 3. 人员资质要求存在倾向性及无关性（中风险）

| 项目 | 内容 |
| :--- | :--- |
| **问题标题** | 项目负责人及团队成员证书要求包含与项目无关的特定证书，且部分证书颁发机构不明 |
| **问题定性** | **中风险** |
| **审查类型** | 评分办法/技术参数倾向性 |
| **原文位置** | 评标信息 技术部分 评分准则 序号 8（项目负责人）及序号 9（团队成员） |
| **原文摘录** | **项目负责人**：<br>"3、具有相关机构颁发的注册信息安全工程师证（CISE）的，得 10 分；<br>4、具有工信部门颁发的高级人工智能应用工程师证书的，得 20 分；<br>6、具有特种设备安全管理和作业人员证书，得 10 分”<br>**团队成员**：<br>"（3）具有相关机构颁发的首席数据官证书，得 5 分；<br>（4）具有相关机构颁发的个人信息保护合规审计师认证证书，得 5 分；” |
| **风险判断** | 1. **特种设备安全管理和作业人员证书**：本项目为软件平台开发，不涉及特种设备（如锅炉、压力容器、电梯等）的制造、安装或维修。要求项目负责人具备此证书与项目履约能力**完全无关**，属于以不合理的条件限制竞争。<br>2. **特定机构证书**：“相关机构颁发的注册信息安全工程师证（CISE）”、“首席数据官证书”、“个人信息保护合规审计师认证证书”等，未明确具体的国家级或行业权威认证机构。若这些证书由特定商业培训机构颁发，则具有明显的指向性，排斥了持有国家软考证书（如信息系统项目管理师、信息安全工程师）的供应商。<br>3. **高级人工智能应用工程师**：虽然项目涉及 AI，但“工信部门颁发”的特定名称证书可能具有排他性，建议改为“具有人工智能相关的高级职称或国家认可的职业技能等级证书”。 |
| **法律/政策依据** | 《中华人民共和国政府采购法实施条例》第二十条第（三）项；《政府采购货物和服务招标投标管理办法》第十七条。 |
| **整改建议** | 1. 删除“特种设备安全管理和作业人员证书”的评分项。<br>2. 将“相关机构颁发”的证书明确为“国家人力资源和社会保障部、工业和信息化部等国家机关或全国性行业协会颁发的……"，或直接改为“具有国家软考高级资格（如信息系统项目管理师、系统架构设计师等）或同等效力的证书”。<br>3. 对于“首席数据官”等新兴证书，若无国家统一标准，建议取消或改为“具有数据治理相关的高级职称或项目经验”。 |

### 4. 业绩要求设置具有排他性（中风险）

| 项目 | 内容 |
| :--- | :--- |
| **问题标题** | 业绩评分中要求“参与过国家相关标准委员会城市运行管理服务相关平台技术研究”及“参与编制深度学习算法相关国家标准” |
| **问题定性** | **中风险** |
| **审查类型** | 评分办法/限制竞争 |
| **原文位置** | 评标信息 商务部分 评分准则 序号 1（投标人相关实力情况评价） |
| **原文摘录** | "1、投标人参与过国家相关标准委员会城市运行管理服务相关平台技术研究的得 20 分；<br>2、投标人参与编制深度学习算法相关国家标准（或行业标准）的得 20 分；” |
| **风险判断** | 1. **排他性极强**：参与国家标准编制或特定委员会技术研究的企业数量极少，通常只有少数几家头部企业或科研院所。将此作为高分值（20 分/项）的评分项，实质上构成了对特定供应商的指向，严重限制了市场竞争。<br>2. **与履约能力关联度弱**：参与标准编制并不直接等同于具备本项目（区级民生诉求平台）的开发和运维能力。 |
| **法律/政策依据** | 《中华人民共和国政府采购法实施条例》第二十条第（三）项：“不得以不合理的条件对供应商实行差别待遇或者歧视待遇……（三）采购需求中的技术、服务等要求指向特定供应商、特定产品……"；《政府采购货物和服务招标投标管理办法》第十七条。 |
| **整改建议** | 1. 删除“参与过国家相关标准委员会……技术研究”及“参与编制深度学习算法相关国家标准”的评分项。<br>2. 改为考察投标人是否具备“类似规模的城市运行管理平台或民生诉求平台开发业绩”，以合同金额和验收报告作为评分依据。 |

### 5. 商务条款中资本背景评分涉嫌歧视（中风险）

| 项目 | 内容 |
| :--- | :--- |
| **问题标题** | 根据股东背景（央企/国企/知名民营）进行评分，涉嫌所有制歧视 |
| **问题定性** | **中风险** |
| **审查类型** | 商务条款/限制竞争 |
| **原文位置** | 评标信息 商务部分 评分准则 序号 2（投标人项目经验情况评价）第 4 点 |
| **原文摘录** | “（4）考察投标人的资本背景与稳定性：对股东为中央企业或省级以上国有投资主体的，得 20 分；股东为知名民营产业资本的，得 10 分。” |
| **风险判断** | 1. **所有制歧视**：政府采购应当遵循公平竞争原则，不得对国有企业、民营企业实行差别待遇。根据股东性质（央企/国企 vs 民营）给予不同分值，直接违反了公平竞争原则，涉嫌排斥民营企业。<br>2. **“知名”定义模糊**：“知名民营产业资本”缺乏客观认定标准，易引发争议。 |
| **法律/政策依据** | 《中华人民共和国政府采购法》第五条：“任何单位和个人不得采用任何方式，阻挠和限制供应商自由进入本地区和本行业的政府采购市场。”；《关于促进政府采购公平竞争优化营商环境的通知》（财库〔2019〕38 号）：“不得设置与采购项目的具体特点和实际需要不相适应的资格、技术、商务条件……不得对国有企业、民营企业实行差别待遇。” |
| **整改建议** | 1. 删除关于股东背景（央企、国企、民营）的评分项。<br>2. 若需考察企业稳定性，可改为“投标人成立年限”或“近三年无重大诉讼/行政处罚”等客观指标。 |

### 6. 注册资本及财务指标设置不合理（中风险）

| 项目 | 内容 |
| :--- | :--- |
| **问题标题** | 设置注册资本及净利润门槛，且与项目规模不匹配 |
| **问题定性** | **中风险** |
| **审查类型** | 商务条款/限制竞争 |
| **原文位置** | 评标信息 商务部分 评分准则 序号 1（投标人相关实力情况评价）第 4、5、6 点 |
| **原文摘录** | "4、投标人的注册资本 200 万以上加 20 分。<br>5、投标人证明其近两年均为盈利状态，且年均营业收入不低于 100 万元，得 20 分。<br>6、投标人证明其近两年均为盈利状态，且年均净利润不低于 50 万元，得 20 分。” |
| **风险判断** | 1. **注册资本限制**：注册资本与企业的实际履约能力无直接必然联系，且 200 万对于软件服务类项目门槛过低，设置此项评分意义不大，且可能被视为以注册资本作为门槛。<br>2. **净利润要求过高**：项目预算约 1293 万元，要求年均净利润不低于 50 万元（且连续两年盈利），对于部分初创型或专注于技术研发的中小企业可能构成不合理限制。虽然作为评分项而非资格条件，但分值过高（20 分）可能导致中小企业处于劣势。<br>3. **重复计分**：营收和净利润均要求“盈利状态”，存在逻辑重叠。 |
| **法律/政策依据** | 《政府采购货物和服务招标投标管理办法》第十七条：“采购人、采购代理机构不得将投标人的注册资本、资产总额、营业收入、从业人员、利润、纳税额等规模条件作为资格要求或者评审因素……"（注：虽然法规禁止作为**资格**要求，但作为**评分**因素若与项目规模严重不匹配，仍可能被认定为不合理条件）。 |
| **整改建议** | 1. 删除“注册资本”评分项。<br>2. 调整财务指标评分，降低分值权重，或改为“提供近三年财务审计报告即可得分”，不再设置具体的营收和净利润门槛，或设置更合理的区间。 |

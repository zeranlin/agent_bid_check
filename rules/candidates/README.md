# 候选规则池

`rules/candidates/` 用于承接外部导入的候选审查点，与 `rules/registry/` 正式规则库物理隔离。

本目录只承担以下职责：

1. 接住外部来源的候选条目。
2. 对候选条目做标准化导入、分流、挂账和快照留档。
3. 为后续 `Task-G7 / Task-G8` 提供稳定底座。

本目录不承担以下职责：

1. 不直接进入正式运行链路。
2. 不直接替代 `rules/registry/`。
3. 不绕过任务单、样本、测试和验收流程。

统一分流结论仅使用以下四类：

- `formal_rule`：可进入正式规则纳管流程
- `conditional_rule`：具备规则价值，但依赖具体画像或场景
- `capability_item`：需要专题能力、抽取能力或结构化理解承接
- `drop`：当前不纳入项目

目录说明：

- `sources/`：来源副本或来源说明
- `imports/`：结构化导入样板
- `mappings/`：候选规则分流台账
- `snapshots/`：导入或分流快照

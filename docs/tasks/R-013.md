# R-013 整改任务单

- 任务编号：`R-013`
- 任务名称：下线“缺失检测报告及认证资质要求”伪规则输出，阻断专题自由发挥误报进入正式风险
- 任务类型：规则整改
- 当前状态：`已通过`

## 1. 背景

在运行 `20260408-162750-07e8e4d3` 中，系统输出了风险点：

- `缺失检测报告及认证资质要求`

M 复核确认：

1. 该风险点不是正式规则库中的 `R-xxx`
2. 来源是 `technical_standard` 专题生成的 `topic::缺失检测报告及认证资质要求`
3. 当前业务规则中，并不存在“只要引用标准，就必须要求 CMA/CNAS 检测报告或 CCC 认证，否则构成正式风险”的通用规则
4. 该条属于专题泛化推断过度，应视为误报来源，不能继续作为正式风险输出

## 2. 整改目标

将“缺失检测报告及认证资质要求”从正式风险输出链路中移除，并建立约束，防止类似无正式规则支撑的专题泛化判断继续进入 `formal_risks`。

## 3. 必须完成项

1. 下线本条风险的正式输出
- 禁止 `topic::缺失检测报告及认证资质要求` 进入 `formal_risks`
- 已有同类标题也不得通过 compare / governance 再次转正

2. 明确输出治理口径
- 对于“缺少检测报告 / CMA / CNAS / CCC”类结论，若无明确正式规则支撑，不得直接输出为正式风险
- 若后续确需保留，最多进入“待补证”，且必须带清晰触发前提；本轮先按“禁出正式风险”处理

3. 拆分错误语义链
- 评分项里的 `CNAS` 证书
- 检测报告里的 `CMA/CNAS` 检测资质
- `CCC` 强制认证要求

这三类语义不得混推，不得因为文中出现或未出现其中一类，就推导另一类风险成立

4. 增加治理保护
- 在输出治理层增加“伪规则 / 未纳管专题标题”拦截机制，避免专题自由发挥文案直接进入正式输出
- 至少对本标题做显式拦截

5. 补测试闭环
- 增加回归测试，覆盖本次柴油文件场景
- 验证修复后该风险点不再出现在正式风险列表
- 不影响已有真正规则命中的认证类风险，如“特定认证证书作为评分条件”等

## 4. 验收标准

满足以下条件才算通过：

1. 本次历史运行对应问题定位清楚，能说明原始来源文件与生成链路
2. `缺失检测报告及认证资质要求` 不再进入正式风险输出
3. 同类“引用标准 -> 自动推断缺失检测报告 / CMA / CNAS / CCC”的泛化路径被拦住
4. 有自动化测试，且测试能稳定覆盖
5. 不误伤已有正式规则：
- 特定认证证书倾向性
- 特定发证机构限制
- 外标引用 / 非进口冲突
- 其他已纳管规则

# R-009 整改任务单

## 基本信息

- 任务编号：`R-009`
- 任务名称：补齐“已取消或非强制资质禁入资格条件”识别
- 任务状态：`已通过`
- 下发对象：`T`
- 监督角色：`M`

## 任务背景

在 `Task-G10` 的真实候选首批优先条目正式纳管结果中，`RC-001` 已被明确建议新增为 `R-009`：

`不得将国务院已明令取消的或国家行政机关非强制的资质、资格作为资格条件`

该问题属于资格条件侧的稳定禁限项，核心风险是：

1. 采购文件将已经取消的资质、资格作为投标准入门槛
2. 或将行政机关非强制的资质、资格作为合格供应商条件
3. 从而形成不当准入限制，限制公平竞争

当前项目中，`R-001 ~ R-008` 尚未覆盖这一类资格条件侧风险，因此需要正式新增 `R-009` 承接。

## 整改目标

让系统能够稳定识别以下风险：

1. 资格条件、合格供应商条件、资格审查表等位置出现资质、资格要求
2. 条款语义命中“已取消资质 / 已取消资格 / 非强制资质 / 非强制资格 / 已明令取消”等表达
3. 相关要求被明确设置为投标准入门槛或资格条件
4. 最终输出标准化正式风险

同时需要避免误报以下场景：

1. 现行法律法规明确要求的法定强制许可或法定准入资质
2. 仅作为政策说明、背景介绍或行业常识说明出现，未形成资格门槛的表述

## 标准风险文案

### 风险标题

`将已取消或非强制资质资格作为资格条件，存在设置不当准入门槛风险`

### 问题定性

`高风险`

### 审查类型

`资格条件合规性 / 不当准入门槛`

### 风险判断要求

1. 条款出现在资格条件、合格供应商条件、资格审查标准、资格审查表等准入相关位置。
2. 条款要求投标人具备某类资质、资格。
3. 该资质、资格属于国务院已明令取消，或属于行政机关非强制性要求。
4. 上述要求被设置为投标前置条件、准入门槛或合格供应商条件。

### 整改建议要求

1. 删除已取消或非强制资质、资格要求。
2. 仅保留与采购标的直接相关、且具有明确法定依据的准入条件。
3. 如确有行业许可要求，应在文件中明确其法定依据，避免扩大设置。

## 规则整改要求

### 1. 资格条件侧信号提取

至少识别以下语义：

- `资格条件`
- `合格供应商条件`
- `资格审查`
- `资格审查表`
- `投标人资格要求`
- `供应商资格要求`

建议形成结构化字段，例如：

- `qualification_requirement_present = true`

### 2. 风险语义提取

至少识别以下语义：

- `已取消资质`
- `已取消资格`
- `已明令取消`
- `非强制资质`
- `非强制资格`
- `行政机关非强制`

建议形成结构化字段，例如：

- `cancelled_or_non_mandatory_qualification_signal = true`

### 3. 准入门槛判断

需要判断上述语义是否被用于：

- `资格条件`
- `准入门槛`
- `合格供应商条件`
- `资格审查通过条件`

建议形成结构化字段，例如：

- `cancelled_or_non_mandatory_qualification_used_as_gate = true`

### 4. Cross-check 规则

当同时满足以下条件时，输出 `R-009` 风险：

1. 出现资格条件侧要求
2. 出现“已取消/非强制资质资格”语义
3. 该语义被明确作为准入门槛

# Task-G15 RC-014 证书暗含经营年限要求补强任务单

- 任务类型：`治理架构实施 / 规则补强`
- 当前状态：`待下发`

- 监督角色：`M`

## 任务背景

`RC-014`：

- 候选原文：`设置证书暗含经营年限要求`

该条不是简单的“某个证书不能要”，而是更隐蔽的倾向性问题：

1. 看似要求证书、认证、资格
2. 实际上通过发证时间、持证年限、连续有效期等表达，变相抬高经营年限门槛
3. 这类问题容易漏报，因为表面上未直接写“经营年限”

当前更适合做成现有规则补强项，而不是立一条孤立新规则。

## 任务目标

1. 补齐“证书要求暗含经营年限门槛”的识别边界
2. 明确其与 `R-006`、`R-010` 等现有证书/评分规则的归属关系
3. 补齐真实表达样本与负样本
4. 输出补强建议包

## 本轮范围

### 聚焦范围

1. 证书需取得满若干年
2. 连续持证满若干年
3. 证书有效期、发证时间、延续记录被用于变相体现企业经营年限

### 本轮不做

1. 不把正常证书有效性要求直接当作经营年限要求
2. 不把法定证书有效期校验误报为倾向性门槛

## 具体整改要求

### 1. 建立隐性年限判定逻辑

至少要识别以下表达模式：

1. 取得证书满 `X` 年
2. 连续持有证书满 `X` 年
3. 发证日期需早于某一时间点
4. 通过证书延续历史、持证时长变相评价经营稳定性

### 2. 明确规则归属

必须说明：

1. 该问题更适合补强 `R-006`
2. 或更适合补强 `R-010`
3. 或需要拆成单独规则

并说明判断原因。

### 3. 样本补强要求

至少补以下样本：

1. 正样本
   - 证书要求暗含企业经营时长或市场进入年限
2. 负样本
   - 正常证书有效期要求
   - 正常证书在有效期内的合规校验
3. 边界样本
   - 发证时间要求较特殊，但是否构成年限门槛仍需结合上下文判断

### 4. 三层落位要求

1. `正式风险`
   - 已能明确看出通过证书条件变相要求经营年限
2. `待补证`
   - 存在年限暗示，但上下文不足
3. `排除`
   - 仅为正常证书有效性、续期或法定合规校验

### 5. 输出要求

本任务结尾必须明确给出：

1. 最终补强归属
2. 推荐补强的 failure code / 规则边界方向
3. 是否需要后续再单独立项

## 交付物要求

1. 隐性年限判定逻辑说明
2. 规则归属建议
3. 正样本、负样本、边界样本
4. 补强建议包
5. 改动文件清单

## M 验收标准

### 通过线

1. 已识别常见隐性年限表达模式
2. 已能区分正式风险、待补证、排除
3. 已给出明确补强归属建议
4. 已补齐真实表达样本与负样本

### 不通过情形

1. 仍把正常有效期校验误判为年限门槛
2. 没有明确归属建议
3. 边界样本缺失
4. 只给抽象描述，没有落到可补强对象

## 任务背景

在真实候选规则分流与首批纳管中，`RC-004`：

- 候选原文：`不得设定特定企业人员证书作为资格条件、评审因素`

其“评审因素侧”已被建议并入 `R-006` 的评分侧补强范围，但“资格条件侧”尚未真正收口。

当前风险是：

1. 后续可能继续出现“特定企业人员证书被作为资格门槛”的漏报
2. 若不拆清资格条件侧与评分因素侧边界，容易重复建设或互相污染
3. 真实文件复核时，难以判断该类条款应落在哪条规则上

因此，本任务聚焦 `资格条件侧`，不重复做评分侧。

## 任务目标

1. 建立 `RC-004` 在资格条件侧的治理边界
2. 明确其与 `R-006`、`R-009 ~ R-012` 的规则分工关系
3. 补齐资格门槛场景下的正样本、负样本、边界样本
4. 明确本条是补强现有规则，还是建议单独立新规则

## 本轮范围

### 聚焦范围

1. 特定企业人员证书被作为资格条件
2. 特定企业人员证书被作为合格供应商条件
3. 特定企业人员证书被作为资格审查通过门槛

### 本轮不做

1. 不重复处理评分因素侧
2. 不把一般项目负责人任职资格直接等同于不当证书门槛
3. 不处理与法定强制持证要求直接对应的场景

## 具体整改要求

### 1. 建立资格条件侧边界说明

至少要回答清楚：

1. 什么叫“特定企业人员证书”被用作资格门槛
2. 什么属于合理履约所需的法定或常见人员资格
3. 什么属于超出履约必要性的特定证书限制
4. 与评分因素侧的边界怎么拆

### 2. 明确规则归属建议

必须给出明确结论：

1. 继续补强现有规则
2. 或建议新开规则

如果建议补强现有规则，必须说明：

1. 补强目标规则是哪条
2. 为什么不单独立项
3. 如何避免和评分侧逻辑混淆

### 3. 样本补强要求

至少补以下样本类型：

1. 正样本
   - 将特定企业人员证书直接作为资格门槛
2. 负样本
   - 合理的项目负责人、现场作业人员法定持证要求
3. 边界样本
   - 证书名称看起来偏强，但可能与特定场景有关

### 4. 三层落位要求

1. `正式风险`
   - 证书要求被明确设置为资格门槛，且缺少履约必要性
2. `待补证`
   - 证书要求偏强，但仍可能存在场景化依据
3. `排除`
   - 证书要求与法定履约、现场作业安全或明确岗位职责直接相关

### 5. 交付结论要求

本任务结尾必须明确给出：

1. 资格条件侧最终归属方案
2. 是否建议进入现有规则正式补强
3. 若建议补强，补到哪条规则
4. 若建议新立项，拟定规则方向

## 交付物要求

1. 资格条件侧边界说明
2. 规则归属建议
3. 正样本、负样本、边界样本清单
4. 三层落位说明
5. 改动文件清单

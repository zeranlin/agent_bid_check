# DOC-001 文档目录治理重构任务单

## 基本信息

- 任务编号：`DOC-001`
- 任务名称：`docs/` 目录治理重构
- 任务类型：`文档治理`
- 当前状态：`已通过`
- 下发对象：`T`
- 监督角色：`M`

## 任务背景

当前仓库中的 `docs/` 目录已经沉淀了大量内容，包括：

- 架构方案
- backlog 文档
- 规则治理文档
- 任务单
- 总台账
- 真实文件复核记录
- 评测与回归说明

这些文档本身有价值，但当前存在明显问题：

1. 顶层混放较多，阅读入口不清晰
2. 治理文档、历史方案、运行台账、复核记录没有按职责分区
3. 新文档应该放哪里、老文档是否已归档，缺少统一规则
4. 后续继续新增规则治理、反馈治理、架构汇报文档时，容易再次混乱

因此需要单独启动一项文档治理任务，把 `docs/` 目录按软件工程思路重新规划，形成清晰、可维护、可扩展的结构。

## 任务目标

1. 给出 `docs/` 目录的正式分层方案
2. 完成现有文档的分类映射与迁移建议
3. 建立统一的文档放置规则与命名规范
4. 提供主入口 README，降低后续查找成本
5. 为后续新增业务功能、治理能力、验收材料预留清晰目录边界

## 具体整改要求

### 1. 输出 `docs/` 目录目标结构

至少明确以下分区是否采用，以及各自职责：

- `docs/architecture/`
- `docs/governance/`
- `docs/trackers/`
- `docs/tasks/`
- `docs/reviews/`
- `docs/reports/`
- `docs/templates/`
- `docs/archive/`

要求：

- 每个目录必须有清晰职责定义
- 明确哪些是“长期有效文档”
- 明确哪些是“阶段性过程文档”
- 明确哪些属于“历史归档”

### 2. 产出现状文档分类清单

至少梳理当前 `docs/` 下主要文档，并说明：

1. 当前路径
2. 文档用途
3. 建议归属目录
4. 是否需要改名
5. 是否建议归档

重点覆盖：

- `v2-remediation-tracker.md`
- `v2-mature-implementation-backlog.md`
- `v2-risk-recall-backlog.md`
- `v2-structure-topic-hardening-backlog.md`
- `v2-regression-hardening-backlog.md`
- `v2-processing-architecture-report.md`
- `szdl2025000495a-review-notes-2026-04-01.md`
- `project-roles-and-issue-closure-v1.md`
- `project-testing-role-and-collaboration-v1.md`
- `docs/governance/*`
- `docs/tasks/*`

### 3. 建立文档放置规范

至少说明：

1. 新架构图文档放哪里
2. 新规则治理文档放哪里
3. 新任务单放哪里
4. 新验收记录或复核记录放哪里
5. 新 backlog 或路线图放哪里
6. 历史已失效方案放哪里

要求：

- 能指导后续所有新增文档
- 避免“什么都放顶层”

### 4. 建立文档索引入口

至少补一份文档入口说明，例如：

- `docs/README.md`

该入口至少应说明：

1. 文档分层结构
2. 常用文档入口
3. 不同角色应该先看哪些文档
4. 新文档接入规则

### 5. 给出迁移实施策略

至少拆成两步：

1. 先补结构与索引，不大规模移动
2. 再按批次迁移老文档并修复链接

要求：

- 第一阶段尽量不影响当前已有使用路径
- 第二阶段要有明确迁移顺序
- 明确哪些文件可以延后迁移

## 交付物要求

T 提交时至少应提供：

1. 一份正式的文档治理方案文档
2. 一份 `docs/README.md` 或等价总入口
3. 一份现状文档分类映射表
4. 一份迁移执行清单
5. 如已实施部分迁移，列出迁移文件清单与链接修复范围

## M 验收标准

### 通过线

- 已形成可执行的 `docs/` 目录治理方案
- 已明确主要文档的归类方式
- 已建立统一入口与后续放置规则
- 方案可直接指导后续规则治理、整改任务和汇报文档归位

### 不通过情形

- 只有原则性描述，没有目录结构
- 没有把现有文档映射到目标目录
- 没有入口 README 或放置规范
- 方案不能指导后续新增文档落位

## T 回复格式

1. 本次定义的目标目录结构
2. 新增或修改了哪些文档
3. 现状文档如何归类
4. 哪些已迁移，哪些暂不迁移
5. 后续新增文档应如何落位
6. 自评是否已达到通过线

# Task-G8 首批高价值候选规则纳管任务单

## 基本信息

- 任务编号：`Task-G8`
- 任务名称：首批高价值候选规则纳管
- 任务类型：`治理架构实施`
- 当前状态：`待执行`
- 下发对象：`T`
- 监督角色：`M`

## 任务背景

完成候选池和快照机制后，下一步不是把全部候选规则一次性纳管，而是从中筛选一批高价值、高确定性、可稳定验证的条目，形成正式纳管入口。

本任务的重点不是“数量多”，而是“第一批纳管规则足够稳”。

如果首批纳管范围失控，后续会直接引入大量误报，反而破坏当前 V2 已有成果。

## 任务目标

1. 从候选池中筛选首批高价值条目
2. 明确哪些条目进入正式规则纳管，哪些暂不进入
3. 建立首批纳管条目与正式规则、任务单、样本、测试之间的映射入口
4. 为后续新增 `R-009+` 或扩展既有规则提供正式治理入口

## 具体整改要求

### 1. 输出首批纳管建议清单

至少输出一份首批纳管建议清单，明确：

1. 建议进入正式规则的候选条目
2. 暂作为条件型规则保留的条目
3. 暂作为能力项保留的条目
4. 删除或暂不纳管的条目

### 2. 明确首批纳管筛选标准

至少明确以下筛选口径：

1. 高频
2. 高业务价值
3. 触发条件稳定
4. 排除条件可定义
5. 可形成统一风险文案
6. 可形成样本与测试闭环

要求：

1. 首批不追求全覆盖
2. 首批范围需控制，避免失控扩张

### 3. 建立首批纳管映射入口

对于建议纳管的首批条目，至少要建立以下映射入口：

1. 映射到现有 `R-xxx`
2. 或映射到拟新增 `R-xxx`
3. 映射到对应任务单
4. 映射到样本和测试状态

要求：

1. 后续能从候选条目反查正式规则或纳管任务
2. 后续也能从正式规则回查候选来源

### 4. 标注现有规则吸收关系

对于已经被现有规则吸收的候选条目，需明确标注，例如：

1. 被 `R-003` 吸收
2. 被 `R-004` 吸收
3. 被 `R-005 / R-008` 吸收

要求：

1. 不允许已经被现有规则吸收的条目再次重复立项
2. 必须避免重复建设

### 5. 形成下一阶段纳管 backlog

除了首批建议纳管项外，还应输出：

1. 第二批候选纳管建议
2. 明显属于能力项的 backlog 建议
3. 暂不纳管但需保留观察的条目说明

## 必须补的测试

### 1. 首批纳管清单存在性测试

断言：

- 首批纳管清单存在
- 清单可读取

### 2. 纳管映射完整性测试

断言：

- 建议纳管条目存在规则编号或拟纳管编号
- 存在任务单或映射入口字段

### 3. 已吸收条目标注测试

断言：

- 已被现有规则覆盖的候选条目有明确吸收标注
- 不会重复进入首批纳管列表

## 交付物要求

T 提交时必须附：

1. 首批纳管建议清单
2. 纳管筛选标准说明
3. 首批条目映射表
4. 已吸收条目标注说明
5. 下一阶段 backlog 建议
6. 测试文件与执行结果

## M 验收标准

### 通过线

- 已形成清晰的首批纳管建议清单
- 首批纳管筛选标准明确
- 建议纳管条目与规则/任务/样本/测试有映射入口
- 已吸收条目未重复立项
- 下一阶段 backlog 有明确说明
- 对应测试通过

### 不通过情形

- 没有筛选标准，只是随意挑几条
- 首批纳管范围过大且无边界控制
- 候选条目与正式规则无法互相追踪
- 已被现有规则覆盖的条目重复进入本轮纳管

# Task-EP2 证据来源分类补齐任务单

## 基本信息

- 任务编号：`Task-EP2`
- 任务名称：证据来源分类补齐
- 任务类型：`治理架构实施 / 重构任务`
- 当前状态：`待下发`
- 下发对象：`T`
- 监督角色：`M`

## 任务背景

`Task-EP1` 完成后，系统已经具备证据层骨架和统一 Evidence 对象。

但如果证据对象还不能稳定判断“这段话来自哪里”，则系统仍会继续出现以下结构性问题：

1. 合同模板、正文条款、留白项、附件说明混在一起
2. 提醒项、需核实项、格式项被误当成正文硬证据
3. 后续问题层与准入层只能在“错来源证据”上继续做判断

因此，`Task-EP2` 的目标是先把证据的“来源类型”做稳定。

本任务不处理“这段话属于技术/商务/评分/资格哪一类业务”，那属于后续 `Task-EP3`。

本任务只回答：

`这段原文到底来自什么类型的证据来源`

## 任务目标

1. 在 Evidence 对象上补齐 `source_kind` 分类能力
2. 让模板条款、留白项、合同格式条款、附件条款、提醒项与正文条款稳定区分
3. 为后续 `problem layer` 和 `risk_admission` 提供更可靠的输入边界
4. 用真实文件验证模板/留白/提醒项类误报是否开始下降

## 本任务范围

本任务负责：

1. `source_kind` 分类体系
2. `source_kind` 分类规则或分类器
3. 证据 trace 中的来源留痕
4. 基础回归与真实文件 replay

本任务不负责：

1. `business_domain` 分类
2. `clause_role` 分类
3. `hard_evidence` 判定
4. 问题层归并
5. 发布层单源化

## 具体整改要求

### 1. 建立 `source_kind` 分类枚举

至少支持以下来源类型：

| source_kind | 含义 |
| --- | --- |
| `body_clause` | 正文条款 |
| `template_clause` | 模板条款 |
| `placeholder_clause` | 留白项 |
| `contract_template` | 合同格式条款 |
| `attachment_clause` | 附件 / 声明 / 承诺 |
| `reminder_clause` | 提醒项 / 待核实项 |
| `form_clause` | 表单 / 格式项 |
| `sample_clause` | 样品相关条款 |

如你实现时发现需要补等价枚举，可补，但不能减少以上能力覆盖。

### 2. 为 Evidence 对象补齐来源分类结果

要求：

1. 每条 Evidence 至少应带 `source_kind`
2. 无法稳定判定时，允许保留兜底类型，但不能空缺
3. 分类结果必须可追溯，后续能解释“为什么判成这个来源类型”

### 3. 优先覆盖的误报高发来源场景

本任务必须优先处理以下高价值场景：

1. 合同模板中的格式条款
2. `____`、`[]`、留空项等模板留白
3. 需人工确认、需进一步核实、建议评估类提醒项
4. 表单、声明函、承诺书等附件或格式文件

### 4. 与准入层的最小联动

本任务不要求你把所有来源类型都直接写进最终 `formal gate` 规则里，但至少要做到：

1. `source_kind` 能被准入层消费
2. 后续 `EP4` 能基于 `source_kind` 做硬证据判定
3. 至少有基础能力证明模板/留白/提醒类来源不会再被当成“无来源信息的正文默认项”

### 5. trace 留痕要求

来源分类必须在 trace 中能看到至少以下信息：

1. 原始摘录
2. 原始位置
3. 最终 `source_kind`
4. 关键判定依据或判定路径

## 必须补的测试

至少补以下测试：

1. `body_clause` 分类测试
2. `template_clause` 分类测试
3. `placeholder_clause` 分类测试
4. `contract_template` 分类测试
5. `attachment_clause` 分类测试
6. `reminder_clause` 分类测试
7. 来源分类结果能进入后续链路测试

## 真实文件验证要求

至少用 2 类真实场景验证：

1. 柴油文件中的模板 / 留白 / 合同格式 / 提醒类误报场景
2. 福建或福州一中真实文件中的类似边界场景

验证目标：

1. 证明 `source_kind` 已真实进入主链路
2. 证明关键误报高发来源能够被稳定识别
3. 证明本任务没有打坏现有正式风险输出

## 交付物要求

至少交付：

1. `source_kind` 枚举或等价定义
2. 来源分类器说明
3. 新增测试清单
4. 真实文件 replay 结果说明
5. 误报前后变化简述

## M 验收标准

### 通过线

满足以下条件可判通过：

1. `source_kind` 分类体系完整
2. Evidence 对象已稳定带有来源分类结果
3. 高价值来源场景已有专项测试
4. 至少 2 类真实文件场景已验证
5. 模板 / 留白 / 提醒类误报有可见改善
6. 本任务未打坏当前已稳定正式风险

### 不通过情形

出现以下任一情况，本任务不能通过：

1. 只是加了枚举，但没有真正进入主链路
2. 只有样本测试，没有真实文件 replay
3. `source_kind` 无法解释分类依据
4. 模板、留白、提醒类高频误报仍无明显改善
5. 本任务引入新的正式风险回退

# Task-OG2 风险归并与分层裁决任务单

## 基本信息

- 任务编号：`Task-OG2`
- 任务名称：风险归并与分层裁决
- 任务类型：`治理架构实施`
- 当前状态：`待下发`
- 下发对象：`T`
- 监督角色：`M`

## 任务背景

当前真实文件复核暴露出的核心问题，已经集中在输出治理层：

- `6/7` 重复报
- `5/14` 重复报
- `15/25` 重复报
- `21/22/23` 同一组项目负责人风险被拆碎
- 该待补证的没降级
- 该排除的没排除

这些问题说明，系统虽然能识别候选风险，但还没有一套稳定的“最终裁决规则”。

`Task-OG2` 的目标，就是把输出治理层里最关键的两件事落地：

1. 风险归并
2. 分层裁决

## 任务目标

1. 建立正式风险的去重与归并机制
2. 建立 `formal / pending / excluded` 三分层裁决器
3. 为模板条款、弱证据、例外口径、重复候选提供统一处理逻辑
4. 用真实文件验证重复报和降级问题已被真正收敛

## 具体整改要求

### 1. 建立风险归并规则

至少支持以下能力：

- 完全重复风险合并
- 近义标题风险归并
- 同一证据组的多专题重复压缩
- 主风险 + 补充说明聚合

本轮必须优先覆盖以下真实问题家族：

- `业绩限定深圳市政府机关 / 特定行政区域`
- `检测 / 相关部门验收费用打包入投标总价`
- `履约保证金 12% 超上限`
- `项目负责人学历 / 职称 / PMP / 分值结构`

### 2. 建立正式分层裁决器

必须建立统一裁决逻辑，将治理结果明确划分为：

- `formal_risks`
- `pending_review_items`
- `excluded_risks`

要求：

- 渲染层不得再次决定分层
- 任何 topic / baseline 原始结果都不能绕过分层裁决器

### 3. 建立降级规则

至少覆盖以下类型：

- 证据不足
- 条款边界不清
- 仅凭“全文未提及”作出的弱判断
- 需要额外确认适用品目或完整政策条款

本轮必须优先覆盖：

- `残疾人福利性单位及监狱企业政策表述不完整`
- `节能环保产品政策条款缺失`
- 其他同类“证据不够硬”的正式风险

### 4. 建立排除规则

至少覆盖以下类型：

- 模板占位条款
- 明显例外口径
- 不应判为正式风险的说明性条款

本轮必须优先覆盖：

- 社保缴纳说明函例外口径
- 已知合同模板占位特征

### 5. 标题单源化

归并后的主风险必须统一标题，不得同一风险家族在同一次结果中出现多个近义标题。

至少要求：

- 一个风险家族只允许一个正式风险主标题
- 其他说明合并为证据或补充说明

## 必须补的测试

### 1. 去重合并测试

断言：

- 同一真实风险家族不会并列输出两条近义正式风险

### 2. 分层裁决测试

断言：

- 弱证据项进入 `pending_review_items`
- 模板/例外项进入 `excluded_risks`
- 只有通过治理裁决的项进入 `formal_risks`

### 3. 真实文件回放测试

至少基于当前柴油文件断言：

- `6/7` 不再重复
- `5/14` 不再重复
- `15/25` 不再重复
- `21/22/23` 已合并
- 指定弱证据项已降级或移出正式风险

## 交付物要求

T 提交时必须附：

1. 风险家族与归并规则说明
2. 分层裁决规则说明
3. 新增去重与分层测试清单
4. 真实文件复跑结果目录
5. 修复前后正式风险数量与差异表

## M 验收标准

### 通过线

- 已建立统一的风险归并机制
- 已建立统一的三分层裁决器
- 真实文件中的重复报明显消失
- 弱证据项不再继续强行进入正式风险
- 对应测试通过

### 不通过情形

- 仍存在同一风险家族并列多条正式风险
- 降级和排除仍靠渲染层临时处理
- 真实文件复跑结果仍与整改前基本一致
- 没有真实文件闭环验证

# W-008 标准规则正式风险误挂人工复核标签整改任务单

## 基本信息

- 任务编号：`W-008`
- 任务名称：修复标准规则型正式风险错误继承“需人工复核”标签的问题
- 任务状态：`已通过`
- 下发对象：`T`
- 监督角色：`M`

## 任务背景

M 在本轮 Web 复核中发现，以下已明确成立的标准规则型正式风险，页面仍展示了 `需人工复核` 标签：

- `将项目验收方案纳入评审因素，违反评审规则合规性要求`

该条目属于已经固化的明确规则型风险：

- 对应规则：`R-003`
- 规则性质：评审规则明示禁止 + 正文评分条款明示纳入
- 正确口径：应直接作为 `正式风险` 输出，不应再挂 `需人工复核`

当前页面仍显示人工复核标签，会削弱结论确定性，影响业务人员对系统可信度的判断。

## 问题定性

这不是新规则缺失，也不是证据不足，而是 `输出层状态污染问题`。

当前系统可能存在以下问题之一：

1. `compare_rule` 标准命中后，仍继承了旧 topic/manual 标志
2. `cluster.need_manual_review` 未在标准规则正式命中后被清理
3. Web 展示层对标准正式风险仍错误显示 `人工复核` 标签

## 整改目标

1. 让标准规则型正式风险不再错误显示 `需人工复核`
2. 保证 `compare_rule` 明确命中且证据充分时，页面标签与实际结论一致
3. 清理标准规则正式风险与人工复核状态混挂的问题
4. 补齐测试与真实文件回放闭环

## 具体整改要求

### 1. 梳理标准规则型正式风险的 manual 判定逻辑

重点检查：

- `comparison.clusters` 中 `need_manual_review` 的生成与继承逻辑
- `compare_rule` 标准规则命中后的优先级
- Web 卡片展示层对 `manual_reasons` / `need_manual_review` 的渲染逻辑

### 2. 明确清理规则

对于以下类型条目：

- 来源包含 `compare_rule`
- 已进入 `formal_risks`
- 对应标准规则已明确命中
- 无真实冲突说明、无证据缺失、无边界不清情形

要求：

- 不得继续展示 `需人工复核`
- 不得继续挂 `人工复核` chip
- 不得继续透传旧的 manual reason

### 3. 重点覆盖本次已发现条目

至少覆盖：

- `将项目验收方案纳入评审因素，违反评审规则合规性要求`

并同步检查：

- `R-001`
- `R-002`
- `R-003`
- `R-004`
- `R-005`
- `R-006`
- `R-007`
- `R-008`

是否存在相同问题。

### 4. 保留必要人工复核边界

不得为了消掉该问题而粗暴取消全部 `manual` 状态。

仍可保留人工复核标签的情形包括：

- 证据缺失
- 规则边界未清
- 不同来源存在真实冲突
- 仅 topic 弱命中、未形成 compare_rule 正式结论

# W-004 综合补强整改单

## 基本信息

- 任务编号：`W-004`
- 任务名称：基于真实文件逐条复核结果，补齐成熟专题的输出、召回、去重与证据质量闭环
- 任务状态：`已通过`

- 监督角色：`M`

## 任务背景

M 已基于真实文件逐条复核当前成熟专题输出：

- 审查对象：`test_target/zf/埋点测试案例和结果/[SZDL2025000495-A]柴油发电机组及相关配套机电设备采购及安装项目.docx`
- 当前结果目录：[20260401-173633-92447616](https://github.com/zeranlin/agent_bid_check/blob/main/data/results/v2/20260401-173633-92447616)
- 逐条复核记录：[szdl2025000495a-review-notes-2026-04-01.md](https://github.com/zeranlin/agent_bid_check/blob/main/docs/reviews/szdl2025000495a-review-notes-2026-04-01.md)

本轮复核结论不是“系统完全不可用”，而是已经明确暴露出 4 类系统性问题：

1. 同一风险被拆成多条近义标题，输出层重复膨胀
2. 证据召回不全时，系统直接下“缺失”类结论
3. 评分项内部满分被误写成评标总分层面的高占比风险
4. 合同模板区、电子化平台说明、公告承接字段等边界场景没有被正确剔除或降级

因此，本单不是单条规则补丁，而是要求 T 针对真实文件复核暴露出的成熟专题系统性缺口，完成一轮综合补强。

## 整改目标

1. 让成熟专题输出更接近真实业务判断，而不是样本泛化判断
2. 让同一组证据在报告中只输出一个主风险，不再重复膨胀
3. 让“证据不足”“待补证复核”“正式风险”三类结果严格分层
4. 让政策类、认证类、公告承接类召回能够覆盖真实文件里的关键条款
5. 让真实文件回放结果中，误报明显下降、重复明显下降、真阳性文案更贴近原文

## 具体整改要求

### 1. 输出层整改

- 合并 `风险点1 / 10 / 17` 为一组主风险：
  - 统一到 `非进口项目中出现国外标准/国外部件相关表述，存在采购政策口径、技术标准口径、验收口径不一致风险`
- 合并 `风险点4 / 14` 为一组主风险：
  - 统一到 `以特定认证及特定发证机构作为评分条件，存在倾向性评分和限制竞争风险`
- 合并 `风险点11 / 22 / 23 / 24` 为一组主风险：
  - 统一到 `评分表达采用定性分档或分点+分档组合，但量化标准、计算方式或判定边界说明不清，存在评审口径不一致风险`
- 合并 `风险点25 / 28` 为一组主风险：
  - 统一到 `项目负责人学历、职称及相关经验被纳入评分因素，需进一步论证其与项目履约能力的直接关联性`
- 保留但必须重写以下条目文案：
  - `风险点3`：必须回到真实原文，如 `验收计划安排`
  - `风险点5`：改为 `潜在费用边界不清/潜在转嫁风险`
  - `风险点15`：改为“评分基线高于需求基线 + 分档陡峭”
  - `风险点21`：聚焦“人员来源被限定为制造商原厂工程师”
  - `风险点30`：改为“验收标准来源表述不清”

### 2. 分值口径整改

- 对所有评分项增加“项内分值 vs 总分权重”判断
- 禁止把评分项内部 `100分` 直接写成：
  - `分值畸高`
  - `占比过大`
  - `决定性因素`
- 重点修复以下条目：
  - `风险点4`
  - `风险点6`
  - `风险点25`
  - `风险点28`
- 若要输出分值风险，必须同时给出：
  - 评分项权重
  - 项内分值
  - 折算后的实际影响判断

### 3. 召回层整改

- 为政策专题补齐固定召回锚点：
  - `关于享受优惠政策的主体及价格扣除比例`
  - `中小企业声明函`
  - `残疾人福利性单位声明函`
  - `监狱企业`
  - `节能产品政府采购`
  - `环境标志产品政府采购`
  - `节能产品认证证书`
- 为认证/证明专题补齐固定召回锚点：
  - `合格证书`
  - `3C认证`
  - `认证合格证`
  - `原产地证明`
  - `三包条例证书`
- 为公告承接型字段补齐识别逻辑：
  - `具体时间详见招标公告`
  - `资格要求详见招标公告`
  - `以公告为准`

### 4. 边界场景整改

- 合同模板区默认不进入成熟专题正式风险：
  - 典型关键词：`合同条款及格式`、`仅供参考`、`合同范本`
  - 解决：`风险点7 / 8 / 27`
- 对以下场景默认不报正式风险：
  - `不接受进口产品` 本身
  - 合理社保例外说明
  - 要求填写品牌、规格和型号
  - 电子化投标场景下声明函不单独签字盖章
  - 解决：`风险点26 / 31 / 33 / 34`
- 对参数类弱信号降级：
  - 有 `允许偏差`、现场空间约束、接口约束的尺寸参数，优先降级为人工复核提示
  - 解决：`风险点16`

### 5. 证据质量拦截整改

- 证据明显截断时，不允许生成正式风险
  - 典型问题：`风险点35`
- 原文摘录为空时，不允许生成正式风险
  - 典型问题：`风险点37`
- 证据未完整覆盖资格条件全文时，不允许生成实体风险标题
  - 改为 `待补证复核项`
  - 典型问题：`风险点36`
- 真实文件输出必须禁止“串样本”：
  - 文件写 `验收计划安排`，报告就不能写成 `验收移交方案`、`验收资料移交安排`
  - 典型问题：`风险点3`

### 6. 规则判断前置条件整改

- 第三方检测类风险必须加前置条件：
  - 只有当文件明确要求专项检测、法定检测、第三方检测时，才检查检测主体和资质
  - 不得因为一般验收条款没写第三方检测就直接报风险
  - 解决：`风险点29`
- 业绩时间窗口风险提高触发门槛：
  - 近三年左右窗口不默认报“时间过短”
  - 解决：`风险点12`
- 付款条款风险必须按完整链路判断：
  - 同时核对预付款、中间款、尾款、试运行期、最长账期
  - 不能只因“试运行三个月后支付20%尾款”就直接上主风险
  - 解决：`风险点32`

# W-007 模板占位条款误报整改任务单

## 基本信息

- 任务编号：`W-007`
- 任务名称：修复合同模板占位条款被误判为正式风险的问题
- 任务状态：`已通过`
- 下发对象：`T`
- 监督角色：`M`

## 任务背景

M 在本轮 Web 复核中发现，系统将以下明显模板占位内容识别为正式风险：

- 原文特征：
  - `甲方收到乙方自测报告后___个工作日内`
  - `检测通过后___个工作日内`
  - `收到甲方整改通知后___个工作日内完成整改`

该类内容具备明显模板特征：

1. 出现 `甲方 / 乙方` 合同范本表述
2. 出现连续下划线、空白占位符、待填写工作日字段
3. 内容位于验收/整改/履约流程模板段落

当前系统却将其提升为“正式风险”，这会直接损害业务可信度。

## 问题定性

这不是一个新的业务风险规则，而是一个明确的 `模板边界误报` 问题。

正确口径应为：

- 模板占位条款默认不进入 `正式风险`
- 如无法完全确认其是否为生效正文，可降为 `待补证复核`
- 对于明显合同范本/模板留白内容，应进入 `已剔除误报`

## 整改目标

1. 识别并拦截合同模板占位内容，不再进入正式风险
2. 对“甲方/乙方 + 下划线留空 + 工作日/期限”组合建立稳定边界规则
3. 让该类内容在最终输出中下沉到 `excluded`，必要时进入 `pending`
4. 补齐真实文件和单元测试闭环，防止后续回流

## 具体整改要求

### 1. 增加模板占位识别规则

至少识别以下信号组合：

- `甲方` / `乙方`
- `____`、`___个工作日`、`空白待填时限`
- `收到通知后`、`检测通过后`、`整改后`、`验收后`
- 合同履约/验收/整改模板语气

要求：

- 当上述信号组合同时出现时，优先判定为 `模板占位内容`
- 不得直接输出为正式风险

### 2. 调整输出分层

分层口径要求：

- 明显模板占位条款：进入 `excluded_risks`
- 边界不清、无法确认是否为生效正文：进入 `pending_review_items`
- 禁止进入 `formal_risks`

### 3. 修复证据文案

对该类内容如保留在 `excluded/pending`，输出文案应调整为：

- `检测到合同/协议模板中的时限占位符，属于模板留白，不直接作为正式风险输出`

不允许继续输出为：

- `重大履约风险`
- `重大验收风险`
- `重大责任失衡风险`

### 4. 真实文件回放校验

必须基于当前真实文件重跑结果校验：

- 该模板占位条款不再进入 `formal_risks`
- 若保留，应仅出现在 `excluded` 或 `pending`

## 必须补的测试

### 1. 单元测试

新增至少 1 条模板占位识别测试，覆盖：

- `甲方/乙方`
- `___个工作日`
- 验收/整改流程模板

断言该类内容不进入正式风险。

### 2. 回放测试

在真实文件回放测试中补断言：

- 上述模板占位条款不再出现在 `formal_risks`

## 交付物要求

T 提交时必须附：

1. 改动文件清单
2. 模板占位识别逻辑说明
3. 新增测试清单
4. 新的真实文件回放结果目录
5. 修复前后对比说明

## M 验收标准

### 通过线

- 模板占位条款不再进入正式风险
- 能稳定识别 `甲方/乙方 + 下划线留空 + 工作日占位` 组合
- 真实文件回放验证通过
- 没有引入新的正文合同条款误伤

### 不通过情形

- 该类条款仍出现在 `formal_risks`
- 仅靠人工解释，未在系统规则中落地
- 通过“全部剔除合同条款”粗暴规避，导致正文合同风险被一起吞掉

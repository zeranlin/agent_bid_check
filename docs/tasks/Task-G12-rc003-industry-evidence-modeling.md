# Task-G12 RC-003 行业化证据边界建模任务单

## 基本信息

- 任务编号：`Task-G12`
- 任务名称：RC-003“资质要求与采购需求不相符”行业化证据边界建模
- 任务类型：`治理架构实施 / 候选规则前置建模`
- 当前状态：`待下发`
- 下发对象：`T`
- 监督角色：`M`

## 任务背景

在真实候选规则首轮分流中，`RC-003`：

- 候选原文：`不得设置与需求不相符的资质要求`

被识别为高业务价值条目，但未直接进入首批正式纳管，核心原因不是价值不够，而是：

1. “与需求不相符”语义过宽
2. 如果没有证据边界模型，容易把“看起来不相关”误当成正式风险
3. 直接全国化纳管，误报概率较高

因此，本轮不直接立 `R-013`，而是先做一张 `行业化证据边界建模单`，围绕当前高频业务场景：

- 柴油发电机组
- 机电安装
- 配套设备采购

先把“什么叫明显错配、什么叫待补证、什么叫合理要求”建稳，再决定是否升级成正式规则。

## 任务目标

1. 为 `RC-003` 建立行业化首批证据边界模型
2. 明确“资质要求与采购需求不相符”的判定字段、判定流程和三层落位规则
3. 补齐正样本、负样本、边界样本
4. 用真实文件或行业样本回放评估误报/漏报风险
5. 最终给出：`是否建议升级为正式规则 R-013` 的结论

## 本轮范围

### 覆盖行业范围

1. 柴油发电机组采购
2. 机电安装
3. 配套设备采购

### 本轮不做

1. 不直接全国化推广到所有行业
2. 不在证据边界未稳前直接输出正式规则
3. 不把“感觉不相关”直接当作正式风险依据

## 具体整改要求

### 1. 建立证据字段模型

至少形成以下结构化字段：

1. `procurement_object`
   - 采购标的
2. `procurement_scene`
   - 采购场景
3. `qualification_name`
   - 资质名称
4. `qualification_usage`
   - 资质用途
5. `demand_relevance`
   - 与履约直接相关 / 间接相关 / 相关性不足
6. `industry_match_level`
   - 行业匹配 / 跨行业疑似错配 / 明显错配
7. `evidence_sufficiency`
   - 证据充分 / 证据一般 / 证据不足
8. `decision_layer`
   - 正式风险 / 待补证 / 排除

### 2. 建立行业化判定口径

T 需要输出一套可复用的判定逻辑，至少回答清楚：

1. 什么样的资质要求可以视为与柴油发电机组/机电安装项目直接相关
2. 什么样的资质要求属于明显跨行业错配
3. 什么情况下只能进入 `待补证`
4. 什么情况下应直接排除，不得报风险

### 3. 明确三层落位规则

#### 正式风险

必须同时满足：

1. 采购标的与行业场景清晰
2. 资质要求明显跨行业或与履约无直接关联
3. 证据足以支撑“需求与资质错配”这一结论

#### 待补证

适用于：

1. 资质要求看起来可疑，但仍可能存在行业特殊依据
2. 当前仅能看出“相关性偏弱”，无法直接判定为明显错配

#### 排除

适用于：

1. 资质要求与采购标的存在合理履约关联
2. 仅因资质名称陌生或字面不直观，不足以认定不相符

### 4. 样本补强要求

至少补以下样本类型：

1. 正样本
   - 明显跨行业错配资质要求
   - 明显与柴油发电机组/机电安装履约无关的资质要求
2. 负样本
   - 与设备采购、安装实施、调试验收直接相关的合理资质要求
   - 法定或行业常见合理资质要求
3. 边界样本
   - 看起来相关性较弱，但不能直接判死的资质要求
   - 需要行业背景或法定依据进一步确认的条目

### 5. 真实回放要求

至少完成以下之一：

1. 用当前柴油发电机组真实文件做针对性回放
2. 补一组行业化样例回放

回放目标：

1. 证明不会把合理资质要求大量误报成正式风险
2. 能区分正式风险、待补证与排除

### 6. 升级判断要求

本任务结尾必须明确给出：

1. 是否建议升级为 `R-013`
2. 如果建议升级：
   - 给出规则名称草案
   - 给出触发条件草案
   - 给出负样本/排除条件草案
   - 给出标准风险文案草案
3. 如果不建议升级：
   - 明确卡点
   - 明确后续还缺什么证据或样本

2. 行业化判定逻辑说明
3. 三层落位规则说明
4. 正样本、负样本、边界样本清单
5. 真实回放或行业回放结果
6. 是否建议升级 `R-013` 的结论说明

## M 验收标准

### 通过线

1. 已形成清晰的证据字段模型，而不是停留在文字描述层
2. 已能区分正式风险、待补证、排除三层
3. 已补齐样本，且边界样本不是空白
4. 已有回放结果证明误报风险可控
5. 已给出是否升级 `R-013` 的明确建议

### 不通过情形

1. 仍以“感觉不相关”代替证据判断
2. 没有负样本和边界样本
3. 不能区分正式风险与待补证
4. 没有真实回放或行业回放支撑
5. 直接主张升级 `R-013`，但没有证据模型支撑

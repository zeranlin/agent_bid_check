## 具体整改要求

### 1. 执行物理迁移

要求：

1. 按 `DOC-001` 的目标目录执行实际移动
2. 迁移时尽量保留原文件名，除非映射表明确建议改名
3. 不要一次引入大量无必要重命名

### 2. 修复引用路径

至少修复以下范围：

1. `docs/` 内部交叉引用
2. `docs/tasks/*`
3. `docs/governance/*`
4. `docs/README.md`
5. `docs/governance/README.md`

如有引用这些文档的其他关键说明文件，也需一并修复。

### 3. 保证入口可用

要求：

1. `docs/README.md` 中的常用入口必须更新到新路径
2. 二级目录 `README.md` 应继续可读
3. 不允许迁移后出现主要入口失效

### 4. 控制变更边界

要求：

1. 本任务以文档目录治理为主，不扩散修改业务代码
2. 不处理与本任务无关的数据目录、结果目录、运行文件
3. 不重写文档内容，只做必要的路径和入口调整

# Task-OG2-F 分层裁决器补强任务单

## 基本信息

- 任务编号：`Task-OG2-F`
- 任务名称：分层裁决器补强
- 任务类型：`治理架构实施 / 补强整改`
- 当前状态：`待下发`
- 下发对象：`T`
- 监督角色：`M`

## 任务背景

M 对 `Task-OG1 ~ Task-OG3` 的首轮验收发现，当前输出治理层虽然已经建立：

- 独立模块
- 治理对象模型
- 真实文件回放测试

但 `OG2` 的核心要求“统一三分层裁决器”还没有完全达成。

当前主要缺口是：

1. `pending_review_items` 和 `excluded_risks` 仍主要来自 `compare.metadata` 透传
2. 输出治理层还不能对这两层做真正的重新裁决
3. 一旦 compare 先分错层，治理层无法完成最终纠偏

因此本补强单的目标，是把输出治理层从“formal 二次治理器”提升为真正的“统一分层裁决器”。

## 任务目标

1. 让输出治理层成为 `formal / pending / excluded` 的唯一最终裁决器
2. 不再简单透传 compare 产出的 pending/excluded 分层
3. 让治理层能够对所有候选对象重新做统一分层判断
4. 为后续误报、漏报、边界条款治理提供稳定承接

## 具体整改要求

### 1. 取消 pending/excluded 的透传式保留

当前治理层对：

- `pending_review_items`
- `excluded_risks`

更多是包装后直接沿用原层级。

整改要求：

- 不得仅按输入 layer 原样保留
- 必须进入统一裁决流程
- 必须由治理规则决定最终目标层

### 2. 建立统一候选输入视图

要求治理层至少支持把以下对象统一视为“待治理候选”：

- compare formal clusters
- compare pending items
- compare excluded items

然后由同一套规则重新裁决最终落层。

### 3. 强化治理规则集

治理规则必须至少能回答：

1. 该项是否应进 `formal_risks`
2. 该项是否应进 `pending_review_items`
3. 该项是否应进 `excluded_risks`
4. 当前输入层级是否应被推翻

### 4. 增加“层级纠偏”测试

至少补以下测试：

- 输入是 pending，但治理层应提升为 formal
- 输入是 excluded，但治理层应提升为 pending
- 输入是 formal，但治理层应下沉为 pending 或 excluded

要求：

- 不能只测“formal -> 降级”
- 必须证明治理层对三层都有最终裁决权

### 5. 真实文件补回放

至少基于当前柴油真实文件补充说明：

- 哪些条目由治理层实际重新裁决
- 哪些不是简单沿用 compare 元数据

## 交付物要求

T 提交时必须附：

1. 分层裁决链路说明
2. 统一候选输入说明
3. 新增层级纠偏测试清单
4. 真实文件回放对照结果
5. 与当前透传式实现的差异说明

## M 验收标准

### 通过线

- 输出治理层已成为真正的唯一分层裁决器
- pending/excluded 不再只是 compare 元数据透传
- 已补齐层级纠偏测试
- 真实文件回放能证明治理层具备重新分层能力

### 不通过情形

- pending/excluded 仍主要保留输入层级
- 治理层只能裁决 formal，不能裁决其他层
- 没有层级纠偏测试

# Task-RA3 提醒项降级与真实回放门禁任务单

## 基本信息

- 任务编号：`Task-RA3`
- 任务名称：提醒项降级与真实回放门禁
- 任务类型：`治理架构实施`
- 当前状态：`待下发`
- 下发对象：`T`
- 监督角色：`M`

## 任务背景

当前正式风险中仍混有一批更像：

- 提醒项
- 证据不足项
- 边界项
- 需要结合业务规模再判断的弱风险

这类问题如果继续停留在 `formal_risks`，会降低客户对正式输出的信任度。

## 任务目标

1. 建立提醒项与证据不足项降级机制
2. 将弱风险从 `formal_risks` 下沉到 `pending_review_items`
3. 把真实文件回放纳入 risk_admission 的正式门禁

## 具体整改要求

### 1. 建立提醒项识别规则

至少覆盖：

- `需确认`
- `需警惕`
- `建议评估`
- `需进一步核实`
- `可能`

默认策略：

- 不得直接进入正式风险

### 2. 建立证据不足项降级规则

典型标题包括：

- 关键条款缺失，需人工复核
- 适用性需进一步核实
- 证据片段未覆盖

默认策略：

- 进入 `pending_review_items`

### 3. 真实文件门禁

至少将以下对象纳入准入回放：

- 福建物业文件
- 当前柴油文件

断言至少包括：

- 已知误报不再进入正式风险
- 已确认硬风险仍稳定保留
- 弱风险 / 提醒项完成降级

### 4. Web / 报告展示对齐

要求：

- 前端和 Markdown 报告只展示准入后的正式风险
- `pending_review_items` 与 `formal_risks` 展示口径清晰分离

## 必须补的测试

1. 提醒项降级测试
2. 证据不足项降级测试
3. 福建文件真实回放测试
4. 柴油文件真实回放测试
5. Web / final output 一致性测试

## 交付物要求

1. 降级规则说明
2. 回放门禁清单
3. 真实文件前后对照
4. 展示层对齐说明

## M 验收标准

### 通过线

- 提醒项和证据不足项已具备稳定降级机制
- 福建文件中弱风险收敛明显
- 柴油文件既有真实风险不被误伤
- Web / 报告与准入结果一致

### 不通过情形

- 提醒项仍直接留在正式风险
- 真实回放未纳入门禁
- 准入层结果与最终展示仍不一致

# Task-RA2 模板与边界证据硬门禁任务单

## 基本信息

- 任务编号：`Task-RA2`
- 任务名称：模板与边界证据硬门禁
- 任务类型：`治理架构实施`
- 当前状态：`待下发`
- 下发对象：`T`
- 监督角色：`M`

## 任务背景

福建文件已经暴露出明确问题：

- 联合体/分包协议模板被拿来支撑正式风险
- 模板类文本与正文条款混合后，误抬正式风险

因此 risk_admission 必须先把“哪些证据天然不能直接支撑正式风险”识别出来。

## 任务目标

1. 建立模板/协议/声明函等证据类型识别
2. 建立模板类文本不得直接进入正式风险的硬门禁
3. 修复福建文件中模板误用导致的高风险误报

## 具体整改要求

### 1. 建立 evidence kind 判定

至少识别：

- `contract_template`
- `declaration_template`
- `joint_venture_template`
- `subcontract_template`
- `optional_form`
- `body_clause`

### 2. 建立硬排除规则

要求：

- 模板/声明函/联合体/分包协议等文本默认不得直接作为正式风险主证据
- 除非有正文硬证据交叉支撑，否则只能进入 `pending_review_items` 或 `excluded_risks`

### 3. 福建文件专项修复

目标：

- 纠正“验收标准模糊且依赖后续合同确定，存在需求条款合规风险”这类由分包协议模板抬起的正式风险
- 至少保证该类问题不再以正式高风险直接输出

### 4. 保留正文硬风险

同时必须保证：

- 品牌倾向性
- 著作权人限制
- 履约后按无效投标处理

这类正文硬风险不被误伤

## 必须补的测试

1. 模板文本识别测试
2. 分包协议不得直接支撑正式风险测试
3. 福建文件模板误报回放测试
4. 正文硬风险保留测试

## 交付物要求

1. evidence kind 规则说明
2. 模板硬门禁规则说明
3. 福建文件回放前后对照
4. 新增测试与结果

## M 验收标准

### 通过线

- 模板/协议/声明函类证据已被识别
- 模板文本不能直接抬正式风险
- 福建文件对应误报已被拦住
- 正文硬风险未被误伤

### 不通过情形

- 仍把模板文本当主证据
- 福建文件误报仍保留为正式风险
- 一刀切拦截，导致正文硬风险也消失

# R-004-F 补强整改任务单

- 任务编号：`R-004-F`
- 关联主任务：`R-004`
- 任务名称：补齐“付款周期短于招标文件要求 / 预付款比例优化”真实表达的评分漏报
- 任务类型：漏报整改 / 规则补强
- 当前状态：待下发
- 提出方：M
- 执行方：T

## 一、问题背景

真实文件 `[SZDL2025000495-A]柴油发电机组及相关配套机电设备采购及安装项目.docx` 中存在如下评分条款：

`评审委员会根据响应情况进行打分，全部满足要求的得80分；在此基础上，付款周期短于招标文件要求或预付款比例更有利于采购人资金安排的，每项加10分，最高加20分。`

M 判定：

- 该条款已直接将付款周期、预付款比例等付款方式内容与加分挂钩
- 应归入 `R-004`“不得将付款方式作为评审因素”规则范围
- 若当前系统未稳定命中，则属于 `R-004` 真实表达漏报

## 二、补强目标

让系统能够稳定识别以下真实表达，并统一输出到 `R-004` 标准风险口径：

- `付款周期短于招标文件要求`
- `预付款比例更有利于采购人资金安排`
- 付款安排与 `加10分 / 最高加20分 / 得分` 等评分表达直接挂钩

## 三、标准输出要求

### 标准风险标题

- `将付款方式纳入评审因素，违反评审规则合规性要求`

### 标准风险文案

- 评分条款将付款周期、预付款比例等付款安排与得分直接挂钩，属于将付款方式作为评审因素，存在评审规则合规风险。
- 付款方式属于合同商务条件，不应作为竞争性评分因素；继续保留该类评分设置，易引发评审争议和差别对待风险。

### 建议整改文案

- 删除“付款周期短于招标文件要求”“预付款比例更有利于采购人资金安排”等加分条件。
- 付款安排应作为统一商务条款约定，不得作为评分项进行优劣比较。
- 评分因素应回归履约能力、技术响应和服务保障能力。

## 四、补强要求

1. 补齐对“付款周期短于招标文件要求”“预付款比例更有利于采购人资金安排”等真实表达的识别。
2. 确保该类表达稳定归入“将付款方式纳入评审因素，违反评审规则合规性要求”。
3. 补充正样本、负样本和真实文件回放测试。
4. 输出层不得只停留在一般商务条款描述，必须明确命中付款方式评分风险。

## 五、样本要求

### 正样本

- 付款周期短于招标文件要求的，每项加分
- 预付款比例更有利于采购人资金安排的，给予加分
- 缩短账期、提高预付款与评分直接挂钩

### 负样本

- 正常合同条款中出现付款周期、预付款，但未用于评分
- 一般付款描述仅用于合同履约，不参与得分
- 明确写为商务响应表内容，但不加分、不打分

## 六、验收标准

1. 真实文件中必须稳定命中该风险。
2. 风险标题、风险文案、整改建议应与 `R-004` 标准口径一致。
3. 不得误伤正常合同付款条款。

## 七、提交要求

1. 列出新增或修改文件。
2. 说明新增了哪些真实表达识别逻辑。
3. 提供样本与测试结果。

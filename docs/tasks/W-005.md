## 当前问题定位

### 1. 合同模板类误报回流

本次正式风险中仍出现以下问题：

- `关键商务条款数据缺失，合同无法执行`
- `履约保证金退还期限未定，存在资金占用风险`
- `验收标准模糊，采购人单方裁量权过大`

这些条目本质上仍来自 `合同条款及格式` 模板区，属于前面已经明确要求排除或降级的内容，不应重新回流到正式风险清单。

### 2. 关键标准风险漏报

本次结果中，以下应命中的关键风险未稳定产出：

- `非进口项目中出现国外标准/国外部件相关表述，存在采购政策口径、技术标准口径、验收口径不一致风险`
- `强制性标准条款未按评审规则标注★，可能导致实质性响应边界不清`
- `验收检测及相关部门验收费用表述笼统，存在费用边界不清和潜在转嫁风险`

这三类问题是该真实文件里最核心、最有业务价值的风险点，当前未命中，说明比较层与专题层的信号汇聚仍不稳定。

### 3. 输出层仍有旧标题回流

本次正式风险里还出现了：

- `评分标准不明确，存在逻辑矛盾`

该标题本应被聚合进：

- `评分表达采用定性分档或分点+分档组合，但量化标准、计算方式或判定边界说明不清，存在评审口径不一致风险`

说明评分类旧标题仍能绕过去重和标题归一规则，造成重复切面回流。

### 4. 技术弱信号仍有表述偏重

本次正式风险中出现：

- `双电源切换柜尺寸允许偏差过大，可能不符合电气安装规范`

但原文是：

- `尺寸:1000*2000*1000（宽*高*厚），允许偏差+10%。`

该类条目按前序复核结论应优先降为人工复核提示，而不是升级成“可能不符合规范”的正式风险。

## 整改目标

1. 杜绝合同模板区误报重新回流到正式风险
2. 让本真实文件中的三条核心标准风险稳定命中
3. 让评分类旧标题不再绕过去重聚合逻辑
4. 让参数弱信号恢复为人工复核提示，不再过度上纲
5. 用本真实文件建立一组可重复回归的验收闭环

## 具体整改要求

### 1. 修复合同模板区误报回流

- 重点治理以下回流条目：
  - `关键商务条款数据缺失，合同无法执行`
  - `履约保证金退还期限未定，存在资金占用风险`
  - `验收标准模糊，采购人单方裁量权过大`
- 检查比较层正式风险装配逻辑，确认是否存在：
  - baseline 风险直接透传
  - 旧标题未命中 `TITLE_CONTRACT_TEMPLATE_RE`
  - 合同专题输出绕过边界降级
- 必须保证：
  - 来自 `合同条款及格式 / 仅供参考 / 合同范本` 的模板型问题，不再进入正式风险
  - 如确需保留，只能进入 `待补证复核项` 或 `已剔除误报`

### 2. 修复关键标准风险漏报

- 必须让以下三类风险在该真实文件中稳定命中：
  1. `非进口项目中出现国外标准/国外部件相关表述，存在采购政策口径、技术标准口径、验收口径不一致风险`
  2. `强制性标准条款未按评审规则标注★，可能导致实质性响应边界不清`
  3. `验收检测及相关部门验收费用表述笼统，存在费用边界不清和潜在转嫁风险`
- 重点排查：
  - `policy + technical` 跨专题聚合是否被新规则压掉
  - `GB252 / BS2869` 星标规则信号是否被 technical 召回遗漏
  - `检测 + 相关部门验收 + 验收合格前` 组合信号是否未进入 compare 规则
- 必须对该真实文件给出新的命中矩阵，逐项说明：
  - 为什么此前没命中
  - 本次修复后靠哪层命中

### 3. 修复评分类旧标题回流

- 本次回流条目：
  - `评分标准不明确，存在逻辑矛盾`
- 要求：
  - 该类标题必须并入“评分表达与量化口径问题”统一标题
  - 不允许在最终正式风险中同时出现：
    - `评分表达采用定性分档...`
    - `评分标准不明确...`
- 检查：
  - 标题归一逻辑
  - 聚合分组 key
  - baseline/专题/compare 标题优先级

### 4. 修复参数弱信号过度升级

- 本次条目：
  - `双电源切换柜尺寸允许偏差过大，可能不符合电气安装规范`
- 要求：
  - 当原文已出现 `允许偏差` 时，默认先降为人工复核提示
  - 除非同时存在：
    - 现场尺寸硬约束
    - 接口不兼容证据
    - 或明确行业规范冲突证据
  - 否则不得直接升级为正式风险

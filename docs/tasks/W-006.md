本轮结论：

2. 但正式风险清单里仍残留一批范围外误报和分层错误
3. 当前系统已经从“核心风险漏报”阶段，进入“真实文件输出收敛和边界治理”阶段

因此，`W-006` 的目标不是再补核心规则，而是把真实文件成熟输出中仍然影响业务可信度的残留误报收干净。

## 本轮验收发现的主要问题

### 1. 合同模板区误报仍有回流

以下条目仍出现在正式风险中，但按既定业务边界不应进入正式风险：

- `付款条款关键数据缺失，无法评估公平性与节点衔接`
- `合同验收时点条款留白，验收流程缺乏明确的可操作性`
- `履约保证金金额及退还期限未明确`
- `验收期限留白，影响付款节点触发`
- `质量保证金及违约责任条款缺失`

这些条目本质上仍来自 `合同条款及格式 / 仅供参考 / 合同范本` 一类模板区域，或属于模板占位字段，不应作为当前招标审查正式风险输出。

### 2. 同一真实问题仍被拆成重复切面

以下条目与已命中的主风险存在重复或错误拆分：

- `标准编号与名称引用混乱且缺失版本信息`
- `燃油标准引用可能涉及废止或滞后版本`

其中：

- “非进口项目 + 外标引用”主风险已经成立
- 当前又额外拆出“版本缺失 / 标准滞后”作为正式风险，容易造成客户误解为多处独立问题
- 尤其 `燃油标准引用可能涉及废止或滞后版本` 的主轴不对，应并回“非进口项目并列外标引用”的一致性风险，不应单独作为正式主风险

### 3. 分值与权重口径仍有表述偏重

以下条目仍带有“高分值 / 关联度存疑”类泛化表述，需要继续收敛：

- `三体系认证设置高分值，需评估与项目履约的关联性`
- `业绩评分内容与采购标的履约能力关联度存疑`
- `项目负责人学历、职称及相关经验被纳入评分因素，需进一步论证其与项目履约能力的直接关联性`

这里不是要求一刀切删除，而是要求：

- 不得把项内 `100分` 继续泛化成总分层面的高影响风险
- 只有当“与履约无直接关联”证据明确，才保留正式风险
- 若仅属于“建议核查”，应降为人工复核提示，而不是正式风险

### 4. 证据不足类条目仍直接进入正式风险

以下条目更适合进入 `待补证复核项`，不应直接上正式风险：

- `具体资格条款缺失，无法判断是否存在排斥性要求`
- `关键人员配置及业绩要求证据缺失，需人工复核`
- `政策导向章节内容缺失，无法确认节能环保等政策落实情况`
- `缺失检测报告及认证要求的具体规定`

问题不在“完全不能提示”，而在于：

- 当前证据并未充分覆盖全文或只是召回不完整
- 系统却直接下了实体风险标题
- 这会把“未召回到”误写成“文件缺失”

### 5. 已确认应剔除的边界场景仍未彻底降级

以下条目仍带有经验泛化色彩，需降级或移出主清单：

- `澄清截止时间未明确填写`
- `采购标的所属行业未明确，影响中小企业声明函填写`
- `人员社保证明要求存在特殊豁免，需防范虚假人员风险`
- `电子投标文件容量限制可能增加投标负担`

这些条目当前缺少足够强的本项目规则依据，不应继续作为正式风险主标题输出。

## 整改目标

1. 让真实文件正式风险只保留高确定性、强业务价值条目
2. 让合同模板区、公告承接字段、召回不足类问题不再混入正式风险
3. 让外标类、评分表达类风险继续去重收敛，不再重复切面输出
4. 让正式风险 / 待补证复核 / 已剔除误报三层边界稳定下来

## 具体整改要求

### 1. 合同模板区继续收口

必须从正式风险中移除以下条目：

- `付款条款关键数据缺失，无法评估公平性与节点衔接`
- `合同验收时点条款留白，验收流程缺乏明确的可操作性`
- `履约保证金金额及退还期限未明确`
- `验收期限留白，影响付款节点触发`
- `质量保证金及违约责任条款缺失`

要求：

- 若证据来自 `合同条款及格式`、`仅供参考`、`合同范本`、合同占位模板区域，默认不得进入正式风险
- 如需保留，只能进入 `已剔除误报` 或单独的模板提示层，不得进入主风险清单

### 2. 外标 / 标准类重复切面归并

必须处理以下重复或错轴条目：

- `标准编号与名称引用混乱且缺失版本信息`
- `燃油标准引用可能涉及废止或滞后版本`

要求：

- 若核心问题本质是“非进口项目并列外标引用”，则统一并入：
  - `非进口项目中出现国外标准/国外部件相关表述，存在采购政策口径、技术标准口径、验收口径不一致风险`
- 不允许对同一组证据再拆出“版本缺失”“标准滞后”等并行正式风险，除非文件另有独立、明确、可验证的标准现行性问题

### 3. 分值 / 权重 / 关联性口径继续收敛

重点复核以下条目：

- `三体系认证设置高分值，需评估与项目履约的关联性`
- `业绩评分内容与采购标的履约能力关联度存疑`
- `项目负责人学历、职称及相关经验被纳入评分因素，需进一步论证其与项目履约能力的直接关联性`

要求：

- 不得因项内 `100分` 自动生成“高分值风险”
- 只有当条款与履约能力的脱钩证据明确，才允许保留正式风险
- 若只是“建议进一步核查关联性”，降为人工复核提示

### 4. 证据不足类必须改为待补证复核

以下条目不得继续留在正式风险：

- `具体资格条款缺失，无法判断是否存在排斥性要求`
- `关键人员配置及业绩要求证据缺失，需人工复核`
- `政策导向章节内容缺失，无法确认节能环保等政策落实情况`
- `缺失检测报告及认证要求的具体规定`

要求：

- 证据不完整、召回未覆盖、只看到承接字段时，不得下正式实体风险
- 应统一进入 `待补证复核项`
- 报告文案必须清楚区分“文件真的缺”与“当前证据未覆盖”

### 5. 边界场景继续降噪

以下条目原则上不再进入正式风险：

- `澄清截止时间未明确填写`
- `采购标的所属行业未明确，影响中小企业声明函填写`
- `人员社保证明要求存在特殊豁免，需防范虚假人员风险`
- `电子投标文件容量限制可能增加投标负担`

要求：

- 没有项目内明确禁止规则、没有高确定性违法依据的，降为提示或剔除
- 不再把经验泛化判断直接写成正式风险

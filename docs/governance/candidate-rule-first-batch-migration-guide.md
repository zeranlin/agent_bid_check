# 首批高价值候选规则纳管建议

## 1. 文档目标

本文档用于承接 `Task-G8` 的首批高价值候选规则纳管建议。

目标是：

1. 从候选池中筛出高确定性、高业务价值条目。
2. 明确哪些候选已被现有规则吸收，避免重复立项。
3. 为后续第二批纳管和能力项建设提供 backlog 入口。

## 2. 首批纳管筛选标准

首批纳管统一使用以下六项标准：

1. 高频
2. 高业务价值
3. 触发条件稳定
4. 排除条件可定义
5. 可形成统一风险文案
6. 可形成样本与测试闭环

边界控制：

1. 首批不追求数量。
2. 首批优先纳入已被现有规则稳定吸收的高价值条目。
3. 条件型规则、能力项和观察项不强行升格为正式规则。

## 3. 首批纳管建议

### 3.1 建议进入正式规则治理入口

当前首批建议纳管对象均已被现有规则吸收，不新增重复立项：

| 候选条目 | 处理方式 | 吸收规则 | 对应任务 |
| --- | --- | --- | --- |
| `CR-001` 不得将项目验收方案作为评审因素 | 已吸收 | `R-003` | `R-003` |
| `CR-005` 付款周期/预付款比例进入评分 | 已吸收 | `R-004` | `R-004-F` |
| `CR-006` 赠送台式电脑、打印机得分 | 已吸收 | `R-008` | `R-008` |
| `CR-007` 通过办公设备配置、会议保障获取评分优势 | 已吸收 | `R-005` | `R-005` |

结论：

1. 这四条具有高业务价值和稳定边界。
2. 但当前已存在稳定正式规则承接，因此本轮不重复新增 `R-009+`。

### 3.2 暂作为条件型规则保留

- `CR-002`
  - 原因：依赖冷链/生鲜等采购画像，暂不作为通用规则启用。

### 3.3 暂作为能力项保留

- `CR-003`
  - 原因：依赖政策章节召回完整性和证据质量判断，适合作为专题能力 backlog。

### 3.4 删除/暂不纳管

- `CR-004`
  - 原因：不属于当前项目规则纳管范围，仅保留观察结论，不进入首批纳管。

## 4. 已吸收条目标注原则

以下情形必须标注“已吸收”，不得重复立项：

1. 候选语义已被现有 `R-xxx` 稳定覆盖。
2. 现有规则已具备任务单、样本、测试和真实回放闭环。
3. 重复新增只会造成治理重复和验收歧义。

## 5. 下一阶段 backlog 建议

### 5.1 第二批候选纳管建议

- 优先从 `conditional_rule` 中挑选画像依赖清晰、负样本可定义的条目
- 当前建议优先观察 `CR-002`

### 5.2 明显属于能力项的 backlog

- 章节召回不全识别
- 政策条款只有标题、缺少正文的证据拦截
- 证据质量不足时的正式风险降级

### 5.3 暂不纳管但保留观察的条目

- `CR-004`
- 其他无法形成统一风险文案或无法定义稳定负样本边界的候选条目

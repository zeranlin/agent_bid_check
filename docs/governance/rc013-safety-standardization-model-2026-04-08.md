# RC-013 安全生产标准化证书行业化边界模型说明

## 1. 目标

本说明用于承接候选规则 `RC-013：不得设置安全生产标准化证书` 的行业化前置建模。

本轮只覆盖：

- 柴油发电机组及机电安装
- 设备采购 + 安装实施混合场景
- 纯货物采购场景

本轮不直接进入 `rules/registry/`，也不直接全国化推广。

## 2. 对象边界

本轮必须明确拆开三类对象：

1. `安全生产标准化证书`
2. `安全生产许可证`
3. 与特定施工实施直接相关的法定安全资格

这里最重要的边界是：

- `安全生产标准化证书` 更接近企业安全管理成熟度或标准化水平证明
- `安全生产许可证` 是施工实施场景下可能直接相关的法定许可
- `特种作业操作证`、`专职安全员安全生产考核合格证书` 等属于施工安全职责直接相关的法定安全资格

因此，不能把“看到安全生产四个字”就统一打成风险。

## 3. 场景化判定逻辑

### 3.1 纯货物采购场景

本轮结论是：

- 在纯货物采购场景下，将 `安全生产标准化证书` 直接设为资格门槛，通常偏强

原因：

- 纯货物供货与企业安全管理成熟度证明之间通常缺少直接履约关联
- 如果没有安装实施、施工改造、吊装接电等内容，证书必要性难以站稳

因此，这类场景更容易进入 `正式风险`。

### 3.2 安装实施或施工改造场景

本轮结论是：

- 在明确施工实施或安全管理职责场景下，应优先识别是否属于 `安全生产许可证` 或法定安全资格
- 不能把这类法定安全要求误报成 `安全生产标准化证书` 风险

也就是说：

- 如果条款要求的是 `安全生产许可证`，通常应落 `排除`
- 如果条款要求的是 `特种作业操作证`、`专职安全员安全考核证` 等法定安全资格，也应落 `排除`

### 3.3 货物+安装混合场景

本轮结论是：

- 混合场景里，`安全生产标准化证书` 可能与项目安全管理存在一定关联
- 但是否足以成为投标准入门槛，通常还不稳

所以这类场景优先落 `待补证`。

## 4. 三层落位规则

### 正式风险

同时满足：

- 场景清楚，且是纯货物采购
- 条款要求的是 `安全生产标准化证书`
- 要求被直接用作资格条件、合格供应商条件或资格审查门槛
- 当前没有安装实施或施工改造依据支撑其必要性

### 待补证

适用于：

- 项目包含安装实施、机房改造、调试接入等混合因素
- 条款要求仍是 `安全生产标准化证书`
- 当前只能看出它“偏强”，但还不能直接定性为明显不当

### 排除

适用于：

- 条款要求的是 `安全生产许可证`
- 条款要求的是与施工实施职责直接相关的法定安全资格
- 当前场景明确属于施工实施、安全管理职责或特种作业场景

## 7. 是否建议升级为正式规则

本轮结论：

- `暂不建议直接升级为正式规则`

原因：

1. 当前行业覆盖仍偏窄
2. 混合场景中的必要性边界仍需更多真实样本验证
3. 真实文件层面暂时只有“误报可控”的辅助核查，缺少稳定的真实正式风险命中样本

当前主要卡点：

- 还缺纯货物采购场景下真实命中的正式风险样本
- 还缺更多工程细分行业的混合场景样本
- 还缺更多可以稳定拆开 `安全生产标准化证书` 与 `安全生产许可证` 的真实文本

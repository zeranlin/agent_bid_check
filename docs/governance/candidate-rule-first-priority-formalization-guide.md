# 真实候选首批优先条目正式纳管建议

## 1. 文档目标

本文档用于承接 `Task-G10`，把 `Task-G9` 产出的 `first_priority_candidates` 收敛成一小批真正可下发的正式纳管入口。

目标是：

1. 不让真实分流结果悬空
2. 不与现有 `R-001 ~ R-008` 重复建设
3. 为下一步 `R-009+` 或现有规则补强提供直接任务入口

## 2. 首批筛选原则

首批统一遵循以下原则：

1. 触发语义明确，尽量是“不得/不得限定/不得要求”类稳定表达
2. 排除条件可以用负样本明确界定
3. 可以形成统一的正式风险标题和整改建议
4. 能与真实文件或真实规则文本闭环
5. 与现有 `R-001 ~ R-008` 治理边界不重复

本轮不优先推进的条目：

1. 需要强知识图谱支撑的宽泛规则
2. 标题仍偏“合理设置/正确设置/需明确”的条件化条目
3. 与现有规则存在明显重叠、但尚未拆清资格侧/评分侧边界的条目

## 3. 首批正式纳管建议

### 3.1 建议新增 `R-009`

- 对应候选：`RC-001`
- 候选原文：`不得将国务院已明令取消的或国家行政机关非强制的资质、资格作为资格条件`
- 建议规则方向：`已取消或非强制资质禁入资格条件识别`

纳管理由：

1. 语义稳定，属于明确禁限项
2. 与现有 `R-001 ~ R-008` 边界不重叠
3. 业务价值高，且容易形成正负样本

### 3.2 建议新增 `R-010`

- 对应候选：`RC-002`
- 候选原文：`不得将国务院已明令取消的资质、资格、认证作为评审因素`
- 建议规则方向：`已取消资质资格认证禁入评分识别`

纳管理由：

1. 与 `R-009` 分别覆盖资格条件侧和评分因素侧
2. 场景稳定，可形成统一风险文案
3. 对真实业务的评审合规价值高

### 3.3 建议新增 `R-011`

- 对应候选：`RC-015`
- 关联候选：`RC-016`
- 候选原文：
  - `不得要求提供有关资质证明文件、证照、证件原件`
  - `不得要求提供电子证照的纸质证照`
- 建议规则方向：`资质证明材料原件及纸质证照要求边界识别`

纳管理由：

1. 两条语义天然成组，适合合并成一个规则治理对象
2. 正反例边界清晰
3. 容易形成统一的资格材料提交合规风险文案

### 3.4 建议新增 `R-012`

- 对应候选：`RC-020`
- 关联候选：`RC-018 / RC-019 / RC-021`
- 候选原文：
  - `不得限定供应商注册地或所在地或要求供应商在某行政区域内设立分支机构。`
  - `不得限定供应商所有制形式`
  - `不得限定供应商组织形式`
  - `不得设定供应商经营网点作为合格供应商资质条款`
- 建议规则方向：`供应商主体身份与地域限制识别`

纳管理由：

1. 这组条目共同指向主体身份/地域限制
2. 作为一组统一规则纳管，比拆成多条单点规则更稳
3. 业务表达常见，后续真实文件复用价值高

### 3.5 建议映射到现有规则补强

- 对应候选：`RC-004`
- 候选原文：`不得设定特定企业人员证书作为资格条件、评审因素`
- 补强归属：`R-006`

补强说明：

1. 其中“作为评审因素”的部分与 `R-006` 的评分倾向性证书/发证机构限制高度相关
2. 本轮建议先把评分侧并入 `R-006` 补强，避免单独再起一条与证书类评分过于接近的新规则
3. 资格条件侧仍需与更广的资质边界条目一起再拆，不建议在本批强行并入

## 4. 暂缓进入首批的高优先条目

- `RC-003`：`不得设置与需求不相符的资质要求`
  - 原因：边界太宽，需要先明确“需求不相符”的判定证据
- `RC-013`：`不得设置安全生产标准化证书`
  - 原因：可能涉及行业/项目画像，不宜脱离场景直接全国化
- `RC-014`：`设置证书暗含经营年限要求`
  - 原因：更适合作为后续证书边界专题的补强项

## 5. 下一步建议

1. 由 `M` 基于本包优先下发 `R-009 ~ R-012` 中的一小批正式整改单
2. 同时安排 `R-006` 的专项补强单，承接 `RC-004` 评分侧
3. 其余高优先但边界偏宽条目继续保留在候选池，不提前硬纳管

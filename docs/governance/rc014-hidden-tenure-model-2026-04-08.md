# RC-014 证书暗含经营年限边界模型说明

## 1. 目标

本说明用于承接候选规则 `RC-014：设置证书暗含经营年限要求` 的补强建模工作。

本轮结论已经固定：

- `最终补强归属：R-006`
- `推荐补强方向：R-006 评分侧证书倾向性边界下新增“隐性年限门槛”子类`
- `当前不建议单独立项`

本轮不把 `R-010` 作为主归属。

## 2. 隐性年限判定逻辑

本问题不是单纯“有没有要求证书”，而是：

- 证书进入评分
- 或证书条件形成倾向性
- 再通过持证时长、发证时间、连续持证历史，变相抬高经营年限或市场进入时长门槛

本轮重点识别以下表达：

1. `取得证书满 X 年`
2. `连续持有证书满 X 年`
3. `发证日期需早于某一时间点`
4. `通过证书延续历史、持证时长变相评价经营稳定性`

## 3. 为什么归属 R-006

`R-006` 的治理对象是评分侧证书/认证倾向性问题。

`RC-014` 的主场景同样是评分侧，只不过问题从“指定某类证书”进一步演化成：

- 不直接写经营年限
- 而是借证书取得时间、连续持证历史、发证日期来实现类似效果

因此它更适合作为 `R-006` 的一个子边界：

- `certificate_tenure_as_hidden_business_age`

不选 `R-010` 的原因是：

- `R-010` 处理的是“已取消/非强制资质资格认证进入评分”
- `RC-014` 处理的是“通过证书历史变相抬高经营年限门槛”
- 两者对象不同，不能混

## 4. 与正常有效性校验的排除边界

这是本单最关键的误报边界。

以下表达必须排除：

- `证书在有效期内`
- `证书未过期`
- `续期有效`
- `法定合规校验`

原因：

- 这些表达只是正常的证书有效性、续期状态或法定合规确认
- 它们不等于通过证书时间历史变相要求企业经营年限

## 5. 三层落位规则

### 正式风险

适用于：

- 已能明确看出通过证书时间条件变相要求经营年限
- 且该要求与评分、加分或倾向性证书条件直接挂钩

### 待补证

适用于：

- 存在年限暗示
- 但上下文不足，仍可能只是特殊校验逻辑或弱提示

### 排除

适用于：

- 仅为正常证书有效性、续期或法定合规校验

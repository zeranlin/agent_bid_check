# RC-003 行业化证据边界模型说明

## 1. 目标

本说明用于承接候选规则 `RC-003：不得设置与需求不相符的资质要求` 的首批行业化建模工作。

本轮只覆盖：

- 柴油发电机组
- 机电安装
- 配套设备采购

本轮不直接进入 `rules/registry/`，也不直接升级为 `R-013`。

## 2. 证据字段模型

本轮统一使用以下结构化字段：

1. `procurement_object`
   采购标的本身，如柴油发电机组、配电柜、机房配套设备。
2. `procurement_scene`
   采购场景，如纯设备采购、安装实施、调试验收、配套改造。
3. `qualification_name`
   文件中要求的企业资质、资格、许可或执业证照名称。
4. `qualification_usage`
   资质的使用方式，如资格条件、合格供应商条件、资格审查依据。
5. `demand_relevance`
   `与履约直接相关 / 间接相关 / 相关性不足`
6. `industry_match_level`
   `行业匹配 / 跨行业疑似错配 / 明显错配`
7. `evidence_sufficiency`
   `证据充分 / 证据一般 / 证据不足`
8. `decision_layer`
   `正式风险 / 待补证 / 排除`

## 3. 行业化判定逻辑

### 3.1 哪些资质可视为直接相关

在柴油发电机组、机电安装、配套设备采购场景中，以下类型通常可以视为与履约直接相关：

- 建筑机电安装工程专业承包资质
- 机电工程专业建造师资格
- 与明确施工实施、电气接入、高处作业等场景直接对应的法定许可或特种作业资格

判定前提不是“资质名字看起来像机电”，而是：

- 采购标的清楚
- 安装实施或调试验收场景清楚
- 资质要求确实服务于履约实施，而不是泛化地抬高门槛

### 3.2 哪些属于明显跨行业错配

以下资质在当前三类行业场景里可作为明显错配样本：

- 农作物种子生产经营许可证
- 旅行社业务经营许可证
- 食品经营许可证
- 出版物经营许可证

这类资质与柴油发电机组交付、设备安装、调试验收之间不存在直接履约关联，且看不出行业特殊依据，可直接作为正式风险候选。

### 3.3 什么情况下只能进入待补证

以下场景先落 `待补证`：

- 看起来偏强，但项目是否包含施工改造并不清楚
- 资质与履约的关系偏弱，但仍可能存在行业特殊依据
- 仅能看出“可能不匹配”，还不能证明“明显错配”

典型例子：

- 安全生产许可证：若只是货物采购则偏强；若包含施工实施则可能合理
- 电子与智能化工程专业承包资质：若包含控制系统集成可能存在关联
- 消防设施工程专业承包资质：若包含消防联动改造则可能有依据

### 3.4 什么情况下应直接排除

以下情形应直接排除，不得报风险：

- 资质要求与采购标的、安装实施、调试验收存在合理履约关联
- 当前只是资质名称陌生、字面看起来不直观，但场景证据支持其合理性
- 当前证据不足以证明“需求与资质错配”

## 4. 三层落位规则

### 正式风险

同时满足：

- 采购标的与采购场景清晰
- 资质要求被当作资格门槛或资格审查依据
- 资质要求明显跨行业或与履约无直接关联
- 证据足以支撑“需求与资质错配”

### 待补证

适用于：

- 相关性偏弱，但还可能存在行业特殊依据
- 场景不完整，不能直接定性为明显错配
- 当前证据只能支撑“可疑”，不能支撑“判死”

### 排除

适用于：

- 资质要求与履约存在合理直接关联
- 仅凭资质名称陌生，不能证明错配
- 当前证据不足以支持风险结论

## 7. 是否建议升级为 R-013

本轮结论：

- `暂不建议直接升级为 R-013`

原因：

1. 当前行业覆盖仍然偏窄，只覆盖了柴油发电机组、机电安装、配套设备采购
2. 边界样本仍集中在施工/货物混合场景，尚未覆盖更多行业
3. 真实文件回放目前主要证明“误报可控”，尚缺足够稳定的真实正式风险样本

如果后续继续推进，建议的 `R-013` 草案可暂记为：

- 规则名称草案：`资质要求与采购需求明显错配`
- 触发条件草案：
  - 采购标的和采购场景可明确识别
  - 资质要求被用作资格条件或资格审查依据
  - 资质要求与履约无直接关联，且存在明显跨行业错配
  - 证据足以支撑正式结论
- 排除条件草案：
  - 与安装实施、调试验收或法定许可存在直接履约关联
  - 当前仅凭资质名称陌生，尚不足以认定不相符
  - 项目场景不清，仍可能存在行业特殊依据
- 标准风险文案草案：
  - `资质要求与采购需求明显错配，存在设置不当准入门槛风险`

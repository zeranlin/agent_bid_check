# V2 第二层与第三层精度提升 Backlog

## 1. 文档定位

本清单用于在当前 `V2` 架构已基本成形的前提下，继续补强：

- 第二层：结构召回与证据聚合
- 第三层：专题深审与细节型风险抽取

目标不是再做一次大重构，而是围绕“更精准找到风险点”持续补：

- 样本厚度
- 召回规则
- 细节型专题能力
- 分层评测与回归闭环


## 2. 当前判断

当前系统成熟度可概括为：

- 第一层全文直审：`基本完成`
- 第二层结构召回：`基本完成，但仍是主要漏召回来源`
- 第三层专题深审：`基本完成，但细节型风险仍有漏抽`
- 第四层汇总补漏：`已完成，当前不是主阻塞点`

因此，下一阶段的主战场明确收敛到：

1. 第二层结构召回补强
2. 第三层专题抽取补强


## 3. 目标与验收口径

### 3.1 总目标

把 `V2` 从“架构正确、评测齐备”继续推进到：

- 复杂文档场景下第二层漏召回更少
- 已召回证据场景下第三层漏抽风险更少
- 对细节型风险形成稳定、可回归的专题能力

### 3.2 验收原则

每个任务必须满足：

- 有明确输入样本
- 有明确输出产物
- 有明确评测方式
- 不破坏当前 `V1`
- 不破坏当前 `V2` 可运行状态

### 3.3 本阶段重点指标

优先盯这几类指标，而不是只看单次效果：

- 第二层：
  - `module_accuracy`
  - `key_recall`
  - `coverage_recall_rate`
  - `mixed_section_secondary_recall_rate`
- 第三层：
  - `topic_hit_rate`
  - `high_medium_hit_rate`
  - `false_positive_rate`
  - `manual_review_expected_rate`
- 跨层：
  - `risk_hit_rate`
  - `miss_rate`


## 4. 第二层 Backlog

### 4.1 目标

- 把第二层继续做成“稳定证据召回层”
- 降低复杂结构文档下的漏召回
- 为第三层提供更完整、更稳定的 `EvidenceBundle`

### 4.2 任务列表

#### Task B5：补强政策类弱结构召回

内容：

- 继续扩 `policy` 相关结构样本
- 覆盖：
  - 前言式政策说明
  - 表格式政策适用说明
  - 政策 + 资格 + 程序混排
  - 非正式标题与无标题政策段

输出：

- `data/examples/v2_structure_eval_samples.json`
- `data/eval/v2_labels/v2_structure_eval_samples.json`

验收：

- `policy` 结构样本数量继续提升
- 新增样本不显著拉低现有结构层指标


#### Task B6：补强技术标准类复杂结构召回

内容：

- 继续扩 `technical_standard` 相关结构样本
- 覆盖：
  - 标准条款与验收条款混排
  - 标准条款在附表中出现
  - 标准编号与评分条件混排
  - 标准说明跨页续写

输出：

- 第二层结构样本补强

验收：

- `technical_standard` 相关 coverage 在新增样本中可稳定通过


#### Task B7：补强业绩人员类结构关注点

内容：

- 把 `performance_staff` 继续从“被 qualification 附带覆盖”推进到“结构上可被稳定观察”
- 覆盖：
  - 团队要求在附录
  - 类似业绩在评分附件
  - 人员与业绩条件和评分条款共享章节

输出：

- 结构层新增 `performance_staff` 样本

验收：

- `performance_staff` 不再只有零散样本
- 与 `qualification/scoring` 的共享稳定性更可观察


#### Task B8：补强强噪声结构场景

内容：

- 专项补：
  - 目录错乱
  - 无标题长段
  - 跨页拼接
  - 表格与正文交替
  - 附录、附表、补遗说明

输出：

- 强噪声样本集增强版

验收：

- `strong_noise` 类样本继续增厚
- 失败原因可更集中暴露真实薄弱点


#### Task B9：压降结构层残留失败类型

内容：

- 针对当前残留失败类型专项处理，优先关注：
  - `secondary_order_mismatch`
  - `primary_order_mismatch`
  - `shared_topic_unstable`
- 只在有固定样本支撑时修规则，不做盲修

输出：

- 第二层规则补丁
- 对应样本回归结果

验收：

- 结构层残留失败类型数量继续下降


#### Task B10：补第二层专项评测说明

内容：

- 把第二层当前关注的高价值样本族、失败原因、回归命令整理成文档
- 形成“第二层该怎么看是否变好”的统一口径

输出：

- `docs/reports/v2-structure-eval-guide.md`

验收：

- 新人可直接按文档回跑第二层评测并理解指标


## 5. 第三层 Backlog

### 5.1 目标

- 把第三层继续做成“细节型风险深审层”
- 降低“章节已召回但风险未抽出”
- 提高对细节型风险的稳定命中率

### 5.2 任务列表

#### Task C5：补标准类细节风险样本

内容：

- 新增 `technical_standard` 专题样本，覆盖：
  - 引用已废止标准
  - 标准名称与编号不一致
  - 引用标准与采购标的不直接相关
  - 检测方法标准与采购要求不匹配

输出：

- `data/examples/v2_topic_eval_samples.json`
- `data/eval/v2_labels/v2_topic_eval_samples.json`

验收：

- 标准类细节风险不再只靠通用 prompt 命中


#### Task C6：补评分类细节风险样本

内容：

- 新增 `scoring` 专题样本，覆盖：
  - 评分档次缺少量化口径
  - 分值裁量空间过大
  - 评分与采购标的关联性不足
  - 评分前后口径不一致

输出：

- 评分专题漏抽样本增强版

验收：

- 评分细节问题在专题层更稳定暴露


#### Task C7：补资格与隐性门槛类样本

内容：

- 新增 `qualification/performance_staff` 专题样本，覆盖：
  - 本地服务机构要求
  - 业绩被抬成资格门槛
  - 人员证书被抬成资格门槛
  - 资格条件与评分条件重复叠加

输出：

- 资格类与人员业绩类专题样本补强

验收：

- 隐性资格门槛场景更稳定命中


#### Task C8：补合同付款与验收失衡类样本

内容：

- 新增 `contract_payment/acceptance` 专题样本，覆盖：
  - 付款与财政资金到位挂钩
  - 付款节点明显偏后
  - 验收裁量过宽
  - 验收与付款条件叠加失衡

输出：

- 商务与付款类漏抽样本增强版

验收：

- 付款与验收类细节风险专题命中更稳


#### Task C9：补“章节已召回但风险未抽出”分型

内容：

- 把第二层已召回但第三层漏抽的问题继续细分为：
  - `evidence_enough_but_risk_missed`
  - `topic_triggered_but_partial_miss`
  - `risk_degraded_to_manual_review`
  - `cross_topic_shared_but_single_topic_hit`

输出：

- 更细的专题失败样本
- 更细的失败原因映射

验收：

- 第二层与第三层之间的断点更容易定位


#### Task C10：补轻量兜底规则的边界治理

内容：

- 对当前专题后处理/轻量兜底规则继续收边界
- 原则：
  - 只兜明显规则性问题
  - 不把兜底做成误报源
  - 每条兜底都要有样本支撑

输出：

- `app/pipelines/v2/topic_review.py` 持续收敛版

验收：

- `topic_hit_rate` 提升时，不显著拉高 `false_positive_rate`


#### Task C11：补第三层专题评测说明

内容：

- 整理第三层当前的专题族、失败原因、评测口径
- 明确哪些问题由专题层主承担，哪些仍需人工复核

输出：

- `docs/reports/v2-topic-eval-guide.md`

验收：

- 第三层调优有统一口径，不再只靠口头记忆

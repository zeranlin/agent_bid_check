# V2 回归达标整改 Backlog

## 1. 目标

本清单用于在 `P2` 交付完成之后，继续把 `V2` 从“评估体系齐备”推进到“回归质量过线”。

当前结论：

- `P2` 交付状态已通过
- 质量门禁未通过
- 阻塞点集中在回归层跨层指标，而不是结构层、专题层、汇总层的单层评估脚本

当前主要问题：

- `structure_hit_rate = 0.50`，低于门槛 `>= 0.80`
- `topic_coverage_hit_rate = 0.6667`，低于门槛 `>= 0.80`
- `risk_hit_rate = 0.6667`，低于门槛 `>= 0.80`
- `miss_rate = 0.3333`，高于门槛 `<= 0.20`

根因收敛：

- 第一层结构识别与证据召回存在漏召回
- 第二层专题覆盖与风险抽取被第一层缺口级联拖累
- 第三层汇总层当前不是主要阻塞点


## 2. 任务拆分原则

- 先补观测与失败定位，再修召回与抽取能力
- 优先围绕固定回归样本闭环，不先做泛化重构
- 每次只修一类失败原因，避免同时引入多重变量
- 每轮修复后必须回跑 `regression` 与 `all eval`
- 不破坏当前 `V1` 与已通过的 `P2` 交付能力


## 3. 阶段总览

建议分为 4 个阶段：

1. `Phase A`：失败样本拆解与观测补强
2. `Phase B`：第一层结构召回修复
3. `Phase C`：第二层专题抽取修复
4. `Phase D`：跨层回归收口与门禁验收


## 4. Phase A：失败样本拆解与观测补强

### 4.1 目标

- 让回归失败不再只有汇总指标
- 能明确定位失败发生在第几层
- 能输出可直接驱动修复的失败原因清单

### 4.2 任务列表

#### Task A1：回归样本分层失败原因展开

内容：

- 在 `scripts/eval_v2_regression.py` 中，为每个回归样本新增：
  - 分层状态
  - 失败原因清单
  - 主阻塞层判断
  - 简短失败摘要

输出：

- `sample.failure_analysis`
- `summary.root_cause_summary`
- `summary.layer_failure_summary`

验收：

- 能区分结构层失败、coverage 失败、专题风险失败
- `regression_structure_gap_001` 可明确标记为“第一层主卡点，第二层级联失败”


#### Task A2：回归失败类型标准化

内容：

- 对失败原因码进行标准化归类：
  - `section_not_found`
  - `module_mismatch`
  - `secondary_modules_missing`
  - `missing_titles`
  - `missing_modules`
  - `risk_not_extracted`
  - `manual_review_flag_mismatch`
  - `false_positive_risk`

输出：

- 统一失败原因字典
- 汇总级原因统计

验收：

- 同一类问题不再出现多种零散表述


#### Task A3：回归失败 Markdown 报告

内容：

- 新增回归失败 Markdown 报告输出
- 每个样本直接展示：
  - 哪一层失败
  - 原因码
  - 原始章节/风险信息
  - 建议修复方向

输出：

- `regression_report.md`

验收：

- 人可直接阅读，不必再手看多个 JSON


## 5. Phase B：第一层结构召回修复

### 5.1 目标

- 把 `structure_hit_rate` 与 `topic_coverage_hit_rate` 拉过门槛
- 优先解决 `scoring` 章节漏召回问题

### 5.2 任务列表

#### Task B1：专项修复 scoring 章节识别

内容：

- 补强对以下标题变体的识别：
  - `评分办法`
  - `评标办法`
  - `评审办法`
  - `综合评分表`
  - `评分细则`

验收：

- 固定回归样本中评分章节不再漏召回


#### Task B2：补强弱结构章节切分

内容：

- 处理：
  - 目录噪声
  - 标题编号漂移
  - 表格与正文混排
  - 长段正文中嵌评分条款

验收：

- 评分相关片段能被稳定切出为候选 section


#### Task B3：补强 scoring 专题 coverage 映射

内容：

- 明确 `scoring` 需要召回的章节与模块
- 支持评分主章节与评分附表共同进入 evidence

验收：

- `topic_coverage_hit_rate` 稳定提升


#### Task B4：新增第一层专项回归样本

内容：

- 增加：
  - 评分章节标题变体样本
  - 评分表混排样本
  - 评分条款藏在商务/技术章节样本

验收：

- 第一层修复不只对单个样本过拟合


## 6. Phase C：第二层专题抽取修复

### 6.1 目标

- 把 `risk_hit_rate` 拉过线
- 把 `miss_rate` 压到门槛以下

### 6.2 任务列表

#### Task C1：专项修复 scoring 专题风险抽取

内容：

- 强化对以下问题的识别：
  - 评分档次缺少量化口径
  - 评分标准不明确
  - 主观分值过高
  - 评分依据与标的关联性不足

验收：

- `评分档次缺少量化口径` 可在回归样本中稳定命中


#### Task C2：补充“章节已召回但风险未抽出”样本

内容：

- 新增专题层负样本与漏抽样本
- 区分：
  - evidence 不足
  - topic 未触发
  - topic 已触发但风险未命中

验收：

- 第二层问题可独立暴露，不再被第一层掩盖


#### Task C3：专题失败原因记录

内容：

- 为专题结果增加失败原因：
  - `missing_evidence`
  - `topic_not_triggered`
  - `risk_not_extracted`
  - `degraded_to_manual_review`

验收：

- 第二层失败能够被机器统计


#### Task C4：明确风险与人工复核边界收紧

内容：

- 减少本应明确输出的风险被降成 `需人工复核`

验收：

- 人工复核误用率下降


## 7. Phase D：回归收口与门禁验收

### 7.1 目标

- 让跨层回归指标整体过线
- 形成下一阶段的可重复验收口径

### 7.2 任务列表

#### Task D1：固定回归跑批脚本

内容：

- 每次修复后固定执行：
  - `scripts/eval_v2_regression.py`
  - `scripts/eval_v2_all.py`

验收：

- 输出可比对、可归档


#### Task D2：门禁阈值固化

内容：

- 以当前门槛为准：
  - `structure_hit_rate >= 0.80`
  - `topic_coverage_hit_rate >= 0.80`
  - `risk_hit_rate >= 0.80`
  - `miss_rate <= 0.20`

验收：

- 总评估报告明确标记是否过门禁


#### Task D3：扩充跨层综合样本

内容：

- 增加 `2-4` 个跨层综合样本，覆盖：
  - 评分链路
  - 技术标准链路
  - 合同付款与验收交叉链路

验收：

- 回归达标不是单样本过拟合

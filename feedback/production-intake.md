# 客户反馈正式运行说明

## 1. 正式接入入口

- 客户反馈记录目录：`feedback/records/`
- 命名规范：`FB-YYYYMMDD-序号-简述.yaml`
- 总台账：`feedback/records/feedback-ledger.md`

## 2. 角色分工

- `O / 客户`：提交真实反馈
- `M`：做初步分析、定性、定任务单、验收
- `T`：修规则、补样本、补测试、跑真实回归

## 3. 标准运行步骤

1. 在 `feedback/records/` 新建反馈记录。
2. 在 `feedback-ledger.md` 追加台账条目。
3. 建立对应任务单，明确归因层级与修复路径。
4. 完成代码修复。
5. 补样本、补测试。
6. 生成真实文件回归结果目录。
7. 更新反馈记录、台账、任务单状态。
8. 交 M 验收。

## 4. 反馈到任务映射

- 新规则缺失：进入规则接入类任务。
- 旧规则补强：进入规则补强类任务。
- 误报抑制：进入输出分层/边界治理类任务。
- 输出层修正：进入 compare / assembler / final output 治理类任务。
